# 隔週Claude Codeメンテナンス 実行

スケジュールタスク `biweekly-claude-maintenance` は第1・第3月曜10:00に定量計測＋Chatwork通知を自動送信する。本コマンドは、その通知を受けた社長が新セッションで呼び出すことで、**5層の最新情報調査＋松竹梅提案＋実行＋結果通知**までをClaude Codeエージェントで完結させる半自動フロー。

---

## 実行手順

### Step 1: Chatwork個人DMから直近の隔週メンテ通知を読込

- `mcp__chatwork__list_room_messages` で room_id `426170119` の直近メッセージを取得
- タイトルに「📊 隔週Claude Codeメンテナンス」を含む最新メッセージを抽出
- 定量計測結果（現状・前回比）を把握

メッセージが見つからない場合は、`management/md-audit/audit_log.csv` の最新行と `management/md-audit/reports/YYYY-MM-DD.md` を読んで代替する。

### Step 2: 5層の最新情報調査（並列エージェント活用推奨）

以下5層を調査する。可能なら `Agent` ツールで並列エージェント起動（`Explore` か `general-purpose`）。

- **層1 公式**: `code.claude.com/docs` / Anthropic Changelog の直近2週間（WebFetch）
- **層2 開発者Tips**: Boris Cherny（Claude Code開発者）のXポスト・ブログ（WebSearch）
- **層3 高評価実例**: GitHub Trending の Claude Code 関連リポジトリ・shanraisshan/claude-code-best-practice 等の更新（WebSearch）
- **層4 コミュニティ事例**: HumanLayer Blog / Level Up Coding / Medium の Claude Code 関連記事直近2週間（WebSearch）
- **層5 事業側変化**: eBay API / BayChat Cowatech側の変更 / GPT・Geminiモデルの新版（WebFetch・WebSearch）

**情報源が読めない場合**は `memory/feedback_fetch_fallback.md` 準拠で諦めず他手段で読む。

### Step 3: 松竹梅提案を生成

調査で得た情報をこのプロジェクトに適用可能な形で整理：

- 🥇 **松**（理想形）: 効果大・工数大。ゴールから逆算したベスト
- 🥈 **竹**（中間）: 現実的な落とし所
- 🥉 **梅**（最低限）: すぐできる暫定対応

各案に以下を明記：
- **工数**（例: 30min / 1h / 3h）
- **効果**（定量可能なら数値。例: ルートCLAUDE.md 77→60行）
- **リスク**（例: 既存フローへの影響・セキュリティ・情報損失）

`memory/feedback_best_first_thinking.md` 準拠で**必ず松をおすすめとして推奨**し、根拠を厚く語る。

### Step 4: セッション内で社長に再提示→判断

社長に以下の形式で提示：

```
📊 隔週メンテ（YYYY-MM-DD）調査完了。以下をご判断ください：

【現状】（Chatwork通知から抜粋）
・ルートCLAUDE.md: 77行
・memory: 34ファイル / 1992行
・settings allow: 78件

【最新動向（2週間）】
・Anthropic公式: ○○○
・Boris Cherny: ○○○
・GitHub Trending: ○○○
・コミュニティ: ○○○
・事業側: ○○○

【提案】
🥇 松: ○○（工数1h・効果大・リスク○）← 推奨
🥈 竹: ○○（工数30min・効果中）
🥉 梅: ○○（工数15min・効果小）

どれを採用しますか？（複数選択可 or 却下）
```

### Step 5: 選ばれた案を実行

- 社長がGOを出した案のみ実装
- 実装中、既存フロー・セキュリティへの影響が見えたら即中断・報告（`memory/feedback_scheduled_tasks.md` ルール1）
- 各実装はタスクごとにgit commit（ハンドオフ原則）
- 影響が大きい変更（CLAUDE.md圧縮・settings変更等）は事前に草案を見せて再確認

### Step 6: 実行結果をChatwork個人DMに再送信

実行完了後、Chatwork個人DM(426170119)に以下を送信：

```
[info][title]✅ 隔週メンテ実行完了｜YYYY-MM-DD[/title]
■ 実行した案
・採用案: ○○○
・コミット: xxxxxxx / yyyyyyy

■ 変更結果（前回比）
・ルートCLAUDE.md: 77→○○行
・memory: 34→○○ファイル
・settings allow: 78→○○件

■ 次の隔週メンテまでの監視ポイント
・○○○
[/info]
```

`memory/feedback_chatwork.md` 準拠の箱型カード形式必須。

### Step 7: 全過程をCampersコンテンツ素材として保存

`education/campers/content-projects/claude-code-maintenance-case-study/session-logs/YYYY-MM-DD.md` に以下を保存：

- 調査結果（5層分・情報源URL付き）
- 提案の松竹梅（工数・効果・リスク）
- 社長判断・採用案
- 実装過程（コミットハッシュ・差分要点）
- 学び・気づき（次回への申し送り）

`memory/feedback_content_recording.md` 準拠。**配信計画・記事公開・X投稿等は社長判断で別途実施、勝手に公開しない**（`memory/feedback_hp_publish_rule.md`・`memory/feedback_slack_rules.md` の精神）。

## 実行しない条件（中断判断）

以下の場合は中断して社長に報告：

- Chatwork DMに「📊 隔週Claude Codeメンテナンス」メッセージが存在しない かつ audit_log.csv も存在しない → スケジュールタスクが動いていない可能性 → `mcp__scheduled-tasks__list_scheduled_tasks` で状態確認
- WebFetch/WebSearch が全て失敗 → ネットワーク・API制限の可能性 → 手段切替（Chromeツール）を試行しても駄目なら中断報告
- 社長が草案に対して「却下」を明示 → Step 5〜7 をスキップして session-log に「判断: 却下」と記録
