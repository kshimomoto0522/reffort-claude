## ルール：「IPブロック」と安易に判断するな

ASICSサイトでAccess Deniedが出たとき、「IPブロック」と判断してブロック解除待ちの監視ループに入るのは**禁止**。

**Why:**
- 2回同じミスを犯した（2026年3月）
- 社長のブラウザからは普通にアクセスできていた → IPブロックではなかった
- 本当の原因は**ヘッドレスFirefoxのBot検出（Akamaiフィンガープリント）**
- IPブロックと誤認し、15分ごとのチェックを延々繰り返して時間を浪費した
- チェック自体が追加アクセスとなり、状況を悪化させた可能性もある

**How to apply:**
1. Access Deniedが出たら、まず「IPブロック」ではなく「Bot検出」を疑う
2. 社長のブラウザで同じURLが開けるか確認する（開ければIP問題ではない）
3. ヘッドレスブラウザ自体の問題なら、ブラウザ設定・フィンガープリント偽装・User-Agent変更などの根本対策を取る
4. 「ブロック解除待ち」の無限ループ監視に入らない
5. 同じチェックを何度も繰り返して時間を浪費しない

## 追加教訓：並列実行の暴走
- 240秒・1台で安定していたのに、120秒・3台並列を試してブロックを招いた
- 安定しているものを急に大幅に変えない。段階的にテストする（まず120秒・1台、次に240秒・2台、のように）
