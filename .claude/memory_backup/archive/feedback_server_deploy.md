type: feedback

さくらVPSへのデプロイで社長に大量の手作業をさせてしまい、結局失敗した。

**Why:** Windows Server + IISのHTTP.sysがカーネルレベルでポート80を占有し、Node.jsでは上書き不能。さくらVPSのネットワークもポート80以外をブロック。事前調査が不十分だった。

**How to apply:**
- サーバーデプロイの前に、必ず①OS種別 ②Webサーバーの有無 ③ポート開放状況 ④SSH可否を確認する
- 社長に手作業をさせるのは最小限にする。「あなたがやってくれ」と言われたら、直接操作できない理由を最初に明確に伝える
- Windows Server + IISの環境にNode.jsを追加するのは避ける。Render/Railway等のPaaSを優先提案する
- 試行錯誤を社長にやらせない。不確実な手順は提案前に代替策を用意しておく
