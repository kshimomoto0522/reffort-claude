# BayChat UI設計原則

## 🎨 配色ルール（最重要）

**BayChatのブランドカラーは紫（purple系）。** モックアップ・テストレンダリング・仕様書の画像すべてで以下を徹底する。

### 許可される色
| 用途 | カラー | 備考 |
|---|---|---|
| プライマリ | `#7c3aed` | メイン紫 |
| プライマリ濃 | `#5b21b6` | 強調・推奨バッジ |
| プライマリ薄 | `#a78bfa` | サブ要素 |
| 背景極薄 | `#f5f3ff` | カード背景 |
| 背景薄 | `#ede9fe` | アクセント背景 |
| テキスト | `#1f2937` | 本文 |
| サブテキスト | `#6b7280` | ラベル |
| 罫線 | `#e5e7eb` | ボーダー |

### ⚠️ 禁止パターン
- **緑（#10b981 等）・赤（#ef4444 等）・黄（#f59e0b 等）を大面積で使う**
- メリットを緑背景、デメリットを赤背景にするような「色で意味を伝える」設計
- low/medium/high を信号機色（緑/黄/赤）で表す

### ✅ 代わりに使う表現
- 意味の区別は**アイコン・記号・ラベル**で行う（◎ △ ★ 推奨バッジ等）
- 強度は**紫の濃淡**で表現（薄→中→濃）
- 複雑度などの段階表現も紫グラデのみ

**Why:** 社長からの指摘（2026-04-15）— 「BayChatは紫のベースカラー以外のカラーは必要最低限におさえシンプルイズザベストを意識しているため、緑や赤や黄色などあると逆にわかりづらい」

## 🌐 英日併記ルール

**バイヤーメッセージ・AI生成返信文は常に英語＋日本語訳の両方を表示する。**

### 理由
- 社長は英語を完璧には読めない
- セラー（日本人ユーザー）も同様の前提
- 実機でも英日併記UIになる見込み（今後の実装方針）

### 実装方針
- バイヤーメッセージ: 受信時に翻訳API（DeepL/Google/Gemini）で即時翻訳、上下または左右並列表示
- AI生成返信文: 既存プロンプト（v2.3）の `buyerLanguage` + `jpnLanguage` 出力をそのまま両方表示
- テスト実行時も必ず両方を表示する（英語だけ出すのはNG）

**Why:** 社長からの指摘（2026-04-15）— 「バイヤーメッセージは英語なのでわたしはすべてを正確に理解できません。今後セラーメッセージ含め、英語と日本語訳を両方表示」

## 🎯 情報設計原則（シンプルイズザベスト）

- **情報量は段階的に増減する**: 最初は多めに出して、多ケーステスト後に削減判断
- **冗長な情報の扱い**: 添付画像が見える文脈なら「写真を添付済み」は冗長になり得る → UX文脈と合わせて判断
- **添付画像は要約画面内でその場で見られるようにする**（別タブに飛ばさない）
- **判断の優先順位**: 画面で一目でわかる要素は要約で再掲しない
