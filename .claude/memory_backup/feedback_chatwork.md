# Chatwork運用ルール

---

## 1. 反応判断：AIへのメンション・返信のみ

AIへのメンション または AIへの返信（スレッド返信）があった場合のみ反応する。
それ以外（社長・スタッフ間のやりとり、画像確認依頼など）はスルー。

**Why:** 関係ないやりとりに首を突っ込まないようにするため。社長が「画像レビューは無視でOK」と言っていたが、それはAI宛てでないから。
**How to apply:** チャットワーク確認時、AI（+REFFORT AI+）へのメンション・返信があるメッセージだけを「対応が必要なもの」として報告する。

## 2. 社長個人DM（room_id 426170119）は箱型カード形式統一

社長個人DM（**room_id 426170119**）への通知は、すべて以下の箱型カード形式で送る。プレーンテキスト禁止。

### 必須フォーマット

```
[info][title]{絵文字} {カテゴリ}｜YYYY-MM-DD[/title]
本文（■で見出し、箇条書き可）[/info]
```

### タイトル行の絵文字プレフィックス（使い分け）
- ✅ 成功・完了報告
- ⚠️ 中断・異常・要注意
- 🚨 エラー・緊急
- 📱 情報ダイジェスト（SNS・ニュース等）
- 📊 レポート・集計
- ❓ 承認待ち・GO待ち

### 実装例
- `[info][title]✅ Campersメンバー削除 完了報告｜2026-04-15[/title]...[/info]`
- `[info][title]📱 X情報ダイジェスト｜2026-04-15[/title]...[/info]`
- `[info][title]❓ Slack返信承認待ち｜2026-04-15[/title]クエットさんから...[/info]`

**Why:** 2026-04-15、社長から「プレーンテキストと箱型カードが混在している。箱型（`[info][title]`）に統一して」と明確な指示。箱型は視認性が高くタイトルで即判別可能、履歴を遡る際にも識別しやすい。
**How to apply:** 社長個人DM（426170119）にpostする全スクリプト・全プロンプトで`[info][title]...[/title]...[/info]`を必ず使用。新規タスク作成時もこのフォーマットをテンプレート化する。グループチャット（【AI】eBay運営 426169912等）は対象外（現状のプレーン運用を維持）。

### 既に対応済み
- `management/x-digest/x_digest.py:171` — [info][title]📱 X情報ダイジェスト｜{today}[/title]
- `education/campers/member-removal-template.md` — 成功時/中断時ともに[info]形式へ更新済み（2026-04-15）

### 今後統一すべき対象
社長個人DM（426170119）にpostする全スクリプト・プロンプト。新規作成時は必ずこの形式で作る。

## 3. Chatwork DMを使わないケース（2026-04-20追加）

**セッション中（リアルタイムでClaudeと会話中）の承認依頼・草案提示は Chatwork DM 不要**。
その場でClaudeがチャット内に直接見せればOK。

Chatwork DM を使うのは以下のケースに限る：
- 社長が帰宅・離席した後に結果を報告する時
- スケジュールタスク・自動処理からの通知
- 非同期で社長の判断を仰ぐ時（社長がPC前にいない前提）

**Why:** 2026-04-20、社長から「今やりとりしてるんだからわたしが帰宅するのでチャットワークに送ってと言わない限り2は不要」と明確な指示。リアルタイム対話中に二重通知は冗長。
**How to apply:** セッション中の草案提示・承認依頼はチャット内で完結。社長から「帰宅する」「チャットワークに送って」等の明示があれば Chatwork DM を使う。
