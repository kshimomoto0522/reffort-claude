# コンテンツ素材の対象者・フレーミング・2軸切り分け

社長が運営する2軸（Campers実名 / 匿名X→Note）の前提を理解した上で素材を整理・記述すること。

**Why:**
- Campersメンバーは**eBayセラー・非エンジニア**。技術詳細を細かく語っても響かない
- 匿名X→Note読者は不特定多数で、技術詳細や失敗譚そのものに価値を感じる層もいる
- BayChat等の自社プロダクト事例は「参考例」止めで、聞き手にBayChatの理解を求めない
- ターゲットを混同して書くと、Campersには難しすぎ・Xには浅すぎ、という両方滑る素材になる

**How to apply:**

### 素材を書く時の基本構造（両軸で再利用可能な形）
1. **eBay運営での効果**を中核に置く（数字Before/After、業務時間短縮、ミス削減 等）
2. **AI活用の具体手順**を続けて書く（コード/設定/プロンプトの実例）
3. **副次例として「他の場面ではこんなこともやれる」**を末尾に挿入（BayChat/Campers運営/直販等）
4. **失敗・遠回り**は別セクションで生のまま残す（後から取捨選択）

### 配信先別の切り出しルール
| 配信先 | 主軸 | 技術詳細 | BayChat言及 | 失敗譚 | 誘導先 |
|---|---|---|---|---|---|
| Campers ウェビナー | eBay運営の効果と実践 | 浅め・概念図で十分 | 「私はBayChatでもこんなことを」程度の参考例（OK） | 概要のみ・教訓に変換 | AIコース（月額追加） |
| Campers グループチャット | 実装の細部Q&A | 必要時に深く | 質問に応じる | 全部生で出してOK | （誘導先なし） |
| 匿名X | 興味喚起のフック | 中程度・スレッド形式 | **❌ 一切出さない**（匿名でも社長・Reffortと特定される） | 失敗譚も人気コンテンツ（BayChat関連は除外） | Note記事 |
| Note 有料記事 | 完結した実装ガイド | 深く・コード込み | **❌ 一切出さない**（同上） | 詳細手順込み（BayChat関連は除外） | （誘導先なし） |

### BayChat 言及ルール（2026-04-29 社長明確化）
- **Campers向けは OK**：「私の場合は自社サービスでもこんなことやった」と参考例として紹介
- **匿名X／Note は ❌ 完全禁止**：匿名でも「あるSaaSプロダクト」と書いた瞬間に Reffort と特定される
- 過去の memory 記載「触れても匿名化」は誤り。**完全禁止が正解**
- 該当素材は `education/campers/content-projects/applications-beyond-ebay/` に Campers 向け参考例専用として保管

### 禁止事項
- Axis 2（匿名X→Note）からCampers/コンサル事業へ誘導すること（社長明示の方針）
- 匿名X記事内で社長の実名・Reffort社名を出すこと
- BayChat内部の機密情報（プロンプト・設計図・スタッフ氏名・Cowatech固有情報）を匿名X/Noteに出すこと
- 「これは記事化しますか？」と社長に確認を取らずに勝手に判断・編集・配信すること

### content-projects/ 配下の素材整理時のコツ（2026-04-29 構造刷新）
- **業務プロセス縦軸（00〜11の12段階）でフォルダ振り分け**：`education/campers/content-projects/by-business-process/`
- 各事例ファイル（cases/*.md）は要点整理して再構成（journey-log.md からの単純コピーではなく）
- 各事例ファイルに**「メンバーへの含意（応用ポイント）」セクション必須**：社長の事例を「テンプレ」ではなく「方法論の例示」として機能させる核心
- 数字Before/After・判断ポイント・学び・配信先別使い分けを毎ファイルに記録
- 軸別の配信時に編集する（口調変更・実名匿名化・誘導先差し替え）
