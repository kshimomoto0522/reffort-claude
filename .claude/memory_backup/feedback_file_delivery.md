ファイルを作成したら、社長に探させず即座にアクセスできるようにする。**ただし開くのは厳選する**。

**Why:** 全ファイルを自動オープンすると大量のウィンドウが開いて閉じる手間が発生する（2026-04-29指摘）。社長が見る必要のない内部ドキュメント更新まで開くのは逆に邪魔。

**How to apply:**

### hookが自動オープンするもの（社長が直接操作する成果物）
- `.env` : 値入力が必要
- `.xlsx` / `.xlsm` / `.pptx` / `.pdf` : Office/PDF成果物

### Claudeが手動判断で開くもの（`start ""`を使う）
以下は**成果物・結果・社長確認が必要な時だけ**開く：
- `.md` : 仕様書・レポート等の成果物 → 開く / CLAUDE.md・memory・index.md等の内部更新 → 開かない
- `.html` : テスト結果・レポート等 → 開く / .preview.html等の中間ファイル → 開かない
- `.csv` / `.txt` : 分析結果・データ出力 → 開く / 作業用中間ファイル → 開かない

### 判断基準
「**社長がこのファイルを今すぐ見る必要があるか？**」→ Yes なら開く、No なら開かない。

### NGパターン（変わらず禁止）
- 「`xxx.md` に保存しました」「ファイルに書き出しました」で終わる（成果物なら開く）
- 「パスは〇〇です」だけ伝える（成果物なら開く）
- 「必要なら開きます」と受け身になる（成果物なら最初から開く）

### .md を社長に見せる時はHTMLに変換して開く
社長はプログラマーではないため、マークダウン記号のままでは読みづらい。成果物の.mdを開く時はHTMLに変換してブラウザで開く。

### AI出力JSON・テスト結果は必ずUIレンダリング
生JSONを社長に見せるのはNG。テスト結果やAI出力はHTMLレンダリングして開く。
