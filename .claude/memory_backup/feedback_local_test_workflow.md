ダイレクト販売ツール（direct-sales）の修正・デプロイは必ず以下のフローで行う：

1. コード修正
2. `node sync-from-prod.js` で本番データをローカルに同期
3. `node server.js` でローカルサーバー起動
4. 社長に http://localhost:3000 で動作確認してもらう
5. OKなら社長指定のタイミングで `git subtree push` で本番反映
6. ローカルサーバー停止

**Why:** バイヤー・スタッフが利用中に画面が変わったり使えなくなったりする事故を防ぐため。2026-04-10にパスワードリセット・データ消失事故が複数回発生し、社長から厳しい指摘を受けた。

**How to apply:**
- 営業時間中の本番pushは禁止
- 修正がどんなに小さくてもローカルで確認してから本番push
- 本番のdata/は絶対に上書きしない（永続ディスクで保護済み）
- bashのcurlで日本語POSTしない（Node.jsスクリプトを使う）
- purchases等のデータ復元時は既存レコードのフォーマットを事前確認する
