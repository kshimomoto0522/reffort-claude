# ルール

## モデル切替
- **モデル選択は社長の判断に委ねる**
- Claudeから「Sonnetに切替えましょうか」「Opusに切替えましょうか」と提案しない
- 使用上限問題は社長が直面した時点で話題にする
- **基本はOpusで進行**する前提

## Effort Level
- デフォルト **High**（`.claude/settings.local.json` の `effortLevel: "high"`）
- 複雑タスク検出時に `ultrathink` 注入で自動Max相当ブースト
- フック実体：`.claude/hooks/effort_booster.py`（UserPromptSubmit）
- ブースト判定キーワードは同ファイル内で管理

## 社長の発言パターン（フック設計の根拠）
- **社長は「軽く」「さっと」「ざっくり」「簡単に」などの速度優先語は使わない** → オプトアウト判定は不要
- **「しっかり」「ちゃんと」「もっと」は品質強調で頻繁に使う** → 即ブースト対象
- 通常の指示では「考えて」「答えて」としか言わない → これらだけではブーストせず、内容のキーワード数で判断

## Why
- 梶谷健人氏の記事（note.com/kajiken0630 2026-04-15公開）「解決法B」を採用
- 社長は使用上限問題にまだ直面していないため、モデル変更は不要
- Effortを自動調整にすることで、毎回`/effort max`を打つ手間を排除
- 簡単な質問にまでMaxを当てると上限到達が早まるので、Highベースライン＋選択的Maxが最適解

## How to apply
- 新セッションでモデル話題を振られない限り、Opus前提で動く
- 社長プロンプトがキーワードに引っかかればMax相当で思考、引っかからなければHighで効率運転
- キーワードを追加・調整したい要望が出たらeffort_booster.pyを編集
