# リサーチツール品質要件

## 5 原則（2026-05-01 社長指摘より）

### 1. 新品/中古の整合性は絶対
**Why:** リサーチ対象が新品（eBay で新品出品）なのに中古商品を仕入候補にしたら、出品時にクレーム不可避。Reffort のセラーアカウントが信用失う。
**How to apply:** 仕入候補のタイトルから「中古」「USED」「ジャンク」「訳あり」等のマーカーを必ず除外。Amazon 等明示マーカーが弱いサイトは、「新品」マーカー（「新品未開封」「正規品」「日本正規品」等）が無いものは慎重に扱う。

### 2. カテゴリ偏重を避ける
**Why:** Reffort のコア事業（スニーカー）に引きずられて視野が狭くなる。日本→米国 eBay 輸出で利益が出るカテゴリは多岐（JDM 自動車パーツ・トレカ・カメラ・腕時計・楽器・釣具・レトロゲーム・万年筆・包丁・コレクター系）。スニーカーはレッドオーシャン化（2026 年現在 47% しか利益出ない）。
**How to apply:** リサーチキーワードは 12 カテゴリ以上を必ず網羅。スニーカーは縮小して 2 銘柄以下。「松（高利益・低競争）」「竹（標準）」「梅（薄利多売）」の 3 層で組む。

### 3. 赤字結果はレポートに含めない
**Why:** 「リサーチ結果」はそのまま出品候補になる前提。赤字を含めるとノイズが増えて意思決定の邪魔になる。
**How to apply:** ゲート機能必須：純利益 ≥ $15 ∧ マージン ≥ 15% ∧ 売れる根拠スコア ≥ 45 点 を満たす候補のみ HTML 出力。全評価結果は CSV/JSON に「ログ」として残す。

### 4. 仕入先は最低 4 サイト並列
**Why:** 楽天と Yahoo!ショッピングだけでは「最安価格」を取りこぼす。Amazon・フリマ・コレクター系（駿河屋）・カメラ家電系（ヨドバシ）も並列で見ないと、利益試算が「実際は楽天よりもっと安い別ECで仕入れられる」を見落とす。
**How to apply:** 楽天・Yahoo!ショッピング・Amazon.co.jp・Yahoo!フリマ・駿河屋を最低限併用。一致スコアで採点した上で最安候補を採用。Sprint 2 でメルカリ・ヨドバシ・AmiAmi・イケベ楽器を追加。

### 5. 「売れる根拠」の複数シグナル提示が必須
**Why:** 社長の言葉「ただ売れているものを探したって売れる根拠ゼロ」「あなたが本当に売れると思う？」。Browse API でアクティブ出品が見つかっただけでは「これは出品して売れる」と確信できない。
**How to apply:** 各候補に以下 7 シグナルから合成した売れる根拠スコア（0-100）を必ず付与：
1. eBay 全体の出品総数（市場規模）
2. ユニークセラー数（競争度・釣鐘型評価）
3. 同型番セラー数（複数セラー出品 = 売れている確証）
4. 価格競争力（自分の想定売価 vs 中央値）
5. 出品の長寿性（itemCreationDate）
6. 上位セラーの実績（feedbackScore）
7. watchCount（要 App Check 申請）

スコア 75+ = strong / 60-74 = moderate / 45-59 = weak（候補限界）/ 45 未満 = insufficient（除外）
