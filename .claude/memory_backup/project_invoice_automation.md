# 月次請求書 半自動化ツール

## 場所
- ツール: `commerce/ebay/staff-ops/invoice-automation/`
- スラッシュコマンド: `.claude/commands/月次請求書.md`（Claude Codeが自動認識）

## 起動方法
- `/月次請求書 YYYYMM` （引数省略時は前月）
- ドライラン → 社長OK → `--apply` で書込の2段階フロー

## 自動化される処理
- **本間**: 件名月変更・日付・単価10,000円固定書込
- **清水**: 仕入管理表「報酬管理」シートの月名行→下方向「小計（円）」のAH列値を単価に反映
- **佐々木**（最複雑）:
  - 当月収支表からSKU=S先頭オーダーを抽出（B:T 19列・FORMATTED_VALUEで列ごと型変換）
  - 実績管理表に新月集計行＋明細行＋予備空白行を追記
  - G3/I3末尾に `+R{nsr}/+U{nsr}` を自動追記
  - I3新値で5万切捨・前月分との差で当月対象額を出し×19%でインセンティブ確定
  - 備考`実績{N}万円 × 19%（累計{X,XXX}万円）`を自動生成

## 社長の手作業
- 出品数・リサーチ数（佐々木 W15・W16）は前月値維持→3月分の正しい数字に手修正
- 単価微調整（本間1月の特例11,000円のような月）
- PDF出力・送付

## 2027年度切替
`config.yml` のシートIDと請求書シート名（"〜（2026）"→"〜（2027）"）の差し替えのみ。

## 実行履歴
- 2026-04-28 初版完成・3月分実行（130件・インセンティブ¥152,000）

## **Why**: 毎月の手作業1時間以上を5分のドライラン確認＋ボタン1個に圧縮。複数シート跨ぎ・累計計算・型混在のため自動化要件は重く、Sheets API＋Pythonで実装。

## **How to apply**:
- 月次請求書の話題が出たら `/月次請求書 YYYYMM` 推奨
- 「収支表が完成したら」と社長が言ったタイミングが起動の合図
- ドライラン結果は必ず社長確認を取ってから `--apply`
- 想定外動作時は `commerce/ebay/staff-ops/invoice-automation/logs/` のJSONログとGoogle Sheetsの版履歴で復元
