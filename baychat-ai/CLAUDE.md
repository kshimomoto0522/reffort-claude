## 自動メッセージ機能（詳細）

### 送信タイミング一覧

| タイミング | 内容 |
|----------|------|
| 商品が売れた時 | すぐ〜24時間後まで指定可能。リピーター専用メッセージも設定可能 |
| オファーが成立/拒否した時 | オファー結果に応じて自動送信 |
| 商品を発送した時 | Mark as shippedが対象 |
| 商品が配達された時 | eBay配送状況を反映 |
| 配達完了のN日後 | 1〜90日後まで設定可能 |
| フィードバックをもらった時 | ポジティブ・ニュートラル・ネガティブ別々に設定可能 |
| 営業時間外 | 曜日ごとに設定。定休日はその曜日の24時間。タイムゾーン設定対応 |

### 共通設定・その他
- トラブル（未着・リターン・ケース・Dispute）が発生したオーダーへの自動送信を停止できる
- 対象外バイヤーの個別設定が可能
- インサート機能：バイヤーID・氏名・オーダーID・商品名・追跡番号・配送先住所等を自動挿入

## ユーザー数・売上データ

### 用語定義
- **Active**: 無料期間中ユーザーを含む全登録ユーザー数
- **Inactive**: 無料終了後に有料契約しなかった、または解約済みユーザー数
- **Total Payment**: 月次売上（円）
- **No plan**: その月の無料ユーザー数（Activeの内訳）
- **Termination**: 解約者数

- → API連携完了後にさらに調整が必要になる可能性あり

### モデル選定結論（2026-03-19）

**採用方針: Gemini 2.5 Flash（Google）**

| 比較項目 | GPT-4.1-Mini（現在） | Gemini 2.5 Flash（次期） |
|----------|---------------------|------------------------|
| 生成速度 | 10秒以上かかる場合あり | 1〜2秒（圧倒的に速い） |
| 品質 | 良好 | GPT-5相当（Pro級） |
| 1ユーザー月額（5回/日×30日） | 約¥24 | 約¥15 |
| 実装 | OpenAI（現在対応済み） | Cowatechへ確認・依頼が必要 |

**結論の根拠**
- GPT-5-Mini Priority でテスト → 速度10秒でNG
- Gemini 2.5 Flash は GPT-5 級の品質・速度1〜2秒・コストはGPT-4.1-Miniより安い
- 1ユーザーあたり約¥15/月（AI Reply オプション ¥2,200 に対してコストは問題なし）
- **コストを下げながら品質を上げられる唯一の選択肢**

## 競合分析

### 主要競合

#### ① Bee（Awiiin社）← 現時点で唯一の直接競合
- URL: https://bee-group.net/
- **強み**: すでにAI予測返信を実装済み。統合ツールとして進化中
- **弱み**: 高価格・UIがチープで使いづらい・認知度が低い・会員数は微々たるもの
- **マーケ**: メルマガ・Youtubeでこまめに発信しているが効果は限定的
- **注意点**: 統合ツール化を進めており、無視できない存在

#### ② ネギ（YouTuber・eBay Japan Awards 3年連続受賞）← 近日参入予定
- **概要**: eBay業界で影響力あるYoutuber。昨年（2024年）、類似ツールを**無料で提供する**と発言
- **狙い**: ツール登録者をコンサル等に誘導するビジネスモデル
- **懸念**: 無料＋影響力のある人物のため、認知・ユーザー獲得は速い可能性あり
- **対応策**: BayChatの質の高さ・AIの精度で差別化

#### ③ 未確認の第三者開発者（小耳に挟んだ情報）
- 詳細不明
- BayChatの認知がある中で参入するということはAI全面活用で一気に奪取を狙っている可能性大

#### ④ 3D Sellers（アメリカ）
- 日本人向けではないため現時点では競合として意識しない

### 競合に対するBayChatの戦略
- **シンプルイズザベスト**: 統合ツール化はしない。CSに特化し使いやすさを維持
- **AI Replyの質**: Beeより高品質なAI返信で差別化
- **認知・信頼**: eBay Japan Awards受賞・既存ユーザーの支持を活かす
- **UI/UX**: BeeのチープなUIに対してクリーンで使いやすいUIを維持

## Cowatech連携ルール

### Claudeの役割
- BayChat AI機能の**開発・実装・保守などすべてをサポート**する
- 方向性・方針・実装の最終決定は**社長が行う**。Claudeが決定することはない

### 仕様書ルール
- 新機能・プロンプト変更は必ず**日本語仕様書**を作成してからCowatechに渡す
- 仕様書フォーマット：「機能概要 / 入力データ / 期待する出力 / 注意事項」
- 実装はCowatechが行う前提でアウトプットを出すこと

### Slackでの連絡ルール

- **担当者**: Dang Van Quyet（クエットさん）
- **メンション必須**: @Dang Van Quyet と @Keisuke Shimomoto を必ず含める
- **新議題の作り方**:
  1. 太字のタイトルでスレッドを作成（例: `*Claude Codeを活用した開発*`）
  2. そのスレッド内に返信する形でメンション＋内容を投稿
- **同じ議題が続く場合**: 同じスレッド内で返信
- **言語**: 日本語（将来的に日本語＋ベトナム語の両方になる可能性あり）
- **Slackチャンネル**: `#baychat-ai導入`（チャンネルID: C09KXK26J8G）
- **クエットさんのSlack表示名**: Dang Van Quyet

### ⚠️ Slack送信・返信ルール（厳守）

#### 定期チェック
- **1時間ごと**にSlackをチェックし、Claudeへのメンションがあるメッセージを確認する

#### 社長の承認が必要な場合
- Claude Code環境下でプロジェクトを進める上で、**Cowatechへの新規確認・依頼が必要**と判断した場合
  → Claudeがメッセージ草案を作成し、社長の承認を得てから送信する
- Cowatechからの返信に**方針・方向性の決定を求める内容**が含まれている場合
  → 社長に報告し、承認を得てから回答する

#### 社長の承認が不要な場合（Claudeが自律的に対応）
- Cowatechに問い合わせた内容の返答が**問題なく来た場合** → お礼を言って終了
- 返答がほしい情報と**異なる・不足している場合** → 再度依頼する
- **クエットさんからの技術的な質問**に対して、Claudeが持つ一般的な技術知識で回答できる範囲であれば回答する
  - 適当なことは言わない。確信がある内容のみ回答する
  - URLが開けない・情報が不足しているなど回答が難しい場合は、その旨を伝えて追加情報を求める
  - 回答の精度はClaude Codeとして社長に回答する時と同じ水準を保つ
  - どこまで踏み込むかは今後調整していく（まずはテスト運用）

#### 返信に両方の内容が含まれている場合
- 承認不要な内容（事実確認・お礼・再依頼）にはその場で返答する
- 承認が必要な内容（方針・方向性の決定）については「社長に確認します」と返答し、社長に報告する

#### 催促禁止
- 問い合わせに対する返答がまだない場合でも、**催促は絶対にしない**。次の1時間チェックを待つ
