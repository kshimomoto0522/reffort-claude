# AI Reply オーロラボーダーエフェクト 実装仕様書

**作成日:** 2026年3月24日
**依頼元:** Reffort（下本）
**対象:** BayChat Web アプリ
**優先度:** 高

---

## 概要

AI Reply機能に関連するUI要素に「オーロラボーダー」エフェクトを追加します。
カードやボタンの**外縁**に、紫〜ピンク〜ラベンダー〜マゼンタのグラデーションが回転するアニメーションボーダーを実装してください。

**モックアップ（HTMLプロトタイプ）:**
`ai-reply-glow-final.html` を参照してください。ブラウザで開くと実際のアニメーションを確認できます。

---

## 適用箇所（全4箇所）

### ① 料金プランページ — AI Replyオプションカード

- **対象:** AI Replyオプションのカード全体
- **エフェクト:** カードの外縁にオーロラボーダーが回転
- **注意:** 「追加する」ボタンや「無料期間中」ボタンにはエフェクト不要（白のまま）
- カード内部のデザイン・レイアウトは現状維持

### ② AI Reply アイコン（契約中表示バッジ）

- **対象:** 「AI | Reply」ロゴバッジの外枠
- **エフェクト:** バッジの外縁にオーロラボーダーが回転
- **注意:** バッジ内部（AI部分の白背景 + Reply部分の紫背景）はそのまま

### ③ メッセージ入力欄 — AIアイコン（ツールバー）

- **対象:** ツールバー内の小さな「AI」アイコン
- **エフェクト:** アイコンの外枠にオーロラボーダーが回転
- **注意:** アイコンのサイズが小さいため、ボーダー幅は2px程度で十分

### ④ AI Reply画面 —「返信を生成」ボタン

- **対象:** 「返信を生成」ボタンの外枠
- **エフェクト:** ボタンの外縁にオーロラボーダーが回転
- **注意:** ボタン内部の紫グラデーション・テキストはそのまま

## エフェクトの技術仕様

### カラー（グラデーション）

以下の色が順番に回転するconic-gradient（円錐グラデーション）を使用します。

```
#7c3aed → #8b5cf6 → #c4b5fd → #ddd6fe →
#f0abfc → #e879f9 → #d946ef →
#ff00ff → #e740c4 → #d946ef →
#a855f7 → #7c3aed（最初に戻る）
```

**色の内訳:**
| 色 | カラーコード | 役割 |
|----|------------|------|
| 紫（濃い） | #7c3aed | ベースカラー |
| 紫（明るい） | #8b5cf6, #a855f7 | 紫のバリエーション |
| ラベンダー | #c4b5fd, #ddd6fe | 淡く上品なアクセント |
| ピンク | #f0abfc, #e879f9 | 華やかさ |
| マゼンタ | #d946ef, #ff00ff, #e740c4 | 鮮やかなアクセント |

### アニメーション

- **回転速度:** 3秒で360度（1周）
- **イージング:** linear（一定速度で回転し続ける）
- **ループ:** 無限ループ（infinite）

### ボーダー幅

| 適用箇所 | ボーダー幅 | border-radius |
|---------|----------|---------------|
| ①オプションカード | 3px | 16px（外側）/ 13px（内側カード） |
| ②AI Replyバッジ | 3px | 28px（外側）/ 25px（内側バッジ） |
| ③AIアイコン（小） | 2px | 6px（外側）/ 4px（内側アイコン） |
| ④返信を生成ボタン | 3px | 10px（外側）/ 7px（内側ボタン） |

## 実装方法

### CSS実装（推奨）

画像素材は不要です。CSSのみで実装できます。

#### 基本構造

```html
<!-- 外側のラッパー（オーロラボーダー役） -->
<div class="aurora-border-wrapper">
  <!-- 内側の本体（カード/ボタン/アイコン） -->
  <div class="inner-content">
    ...コンテンツ...
  </div>
</div>
```

#### CSS

```css
/* CSS Houdini (@property) を使用した回転アニメーション */
@property --aurora-angle {
  syntax: '<angle>';
  initial-value: 0deg;
  inherits: false;
}

@keyframes aurora-spin {
  to {
    --aurora-angle: 360deg;
  }
}

/* オーロラボーダーのラッパー */
.aurora-border-wrapper {
  position: relative;
  border-radius: 16px;       /* 適用箇所に応じて変更 */
  padding: 3px;              /* ボーダー幅 */
  overflow: hidden;
  background: conic-gradient(
    from var(--aurora-angle),
    #7c3aed, #8b5cf6, #c4b5fd, #ddd6fe,
    #f0abfc, #e879f9, #d946ef,
    #ff00ff, #e740c4, #d946ef,
    #a855f7, #7c3aed
  );
  animation: aurora-spin 3s linear infinite;
}

/* 内側の本体 */
.inner-content {
  position: relative;
  border-radius: 13px;       /* 外側より少し小さく */
  background: ...;           /* 元の背景色 */
  z-index: 1;
}
```

### @property 非対応ブラウザへのフォールバック

`@property` は Chrome 85+、Edge 85+ で対応しています。
非対応ブラウザでは回転アニメーションが動かず、静的なグラデーションボーダーとして表示されます（機能的に問題なし）。

**もし全ブラウザ対応が必要な場合:**
JavaScriptで `requestAnimationFrame` を使ってCSS変数を更新する方法でも実装可能です。

---

## 注意事項

- エフェクトは**外縁のみ**に適用。内部のデザイン・レイアウト・カラーは一切変更しないでください
- パフォーマンス: `will-change: background` や `transform: translateZ(0)` でGPUアクセラレーションを活用してください
- モバイル表示でもエフェクトが崩れないよう、レスポンシブ対応を確認してください
- カードの既存のbox-shadowがある場合は、オーロラボーダーと共存させてください
