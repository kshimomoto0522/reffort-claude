# 月次請求書 半自動化

## 概要

eBay外注3名（本間・清水・佐々木）の毎月請求書作成を半自動化するツール。
社長が請求書のPDF化・最終チェックだけ行えばよい状態まで自動でセットする。

## 通常の使い方（社長向け）

Claude Codeで：

```
/月次請求書 202603
```

1. Claudeが **ドライラン** を実行 → 書込予定の値を画面表示
2. 社長が内容を確認 → 「OK」と返す
3. Claudeが **実書込** を実行 → 3シートを開いて完了報告
4. 社長は内容確認・必要に応じて単価微調整・PDF出力

引数を省略すると **前月分** が対象になります。

## 自動化の範囲

| 担当 | 自動 | 手動 |
|---|---|---|
| 本間 | 件名月変更・日付・単価10,000円書込 | （特例月の単価変更時のみ） |
| 清水 | 仕入管理表AH列から単価自動取得・件名月・日付 | （特例時のみ） |
| 佐々木 | 当月SKU=S明細を実績管理表に追記・累計式更新・インセンティブ自動算出・件名月・日付・備考生成 | 出品数・リサーチ数（W15・W16）は前月値維持→社長手修正 |

## 直接コマンド実行（管理用）

```bash
cd commerce/ebay/staff-ops/invoice-automation
python run.py --month 202603              # ドライラン
python run.py --month 202603 --apply      # 実書込
python run.py --month 202603 --targets honma  # 一部のみ
```

## 設定変更（2027年度切替時など）

`config.yml` のスプレッドシートIDだけ差し替え。シート名（`本間請求書（2026）` 等）も年度変更時は更新。

## トラブルシュート

### 「当月の収支表シート『収支表YYYYMM』が見つかりません」
収支表（KeiS）にその月のシートが未作成。社長が収支表完成後に再実行。

### 「報酬管理シートに『X月』が見つかりません」
仕入管理表で当月の月名行（B列）がまだ追加されていない。仕入管理表更新後に再実行。

### 「書込予定範囲に既存データあり」
実績管理表の新月集計行範囲に何か入っている。手動確認のうえ、再実行前に該当行をクリアするか、前月集計行のSUM範囲を調整。

### 失敗時のロールバック
書込前に各シートの該当行範囲を手動でバックアップする運用を推奨（Google Sheetsの版履歴からも復元可）。
ログは `logs/YYYYMM_apply_YYYYMMDD_HHMMSS.json` に書込内容が記録される。
