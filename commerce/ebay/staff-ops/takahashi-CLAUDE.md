## Git操作ルール

### やっていいこと
```bash
git status        # 現在の状態を確認
git add ファイル名  # 変更を記録の準備
git commit -m "内容"  # 変更を記録
git pull          # 最新の状態を取得
git log           # 履歴を見る
```

### やり方（Pull Requestベース）
1. 作業ブランチを作る: `git checkout -b takahashi/作業名`
2. 作業して、こまめにcommitする
3. 完了したら: `git push origin takahashi/作業名`
4. GitHubでPull Requestを作成
5. 社長がレビュー→OKならマージ

### 絶対にやらないこと
- `main` ブランチに直接pushしない
- `git push --force` は使わない
- `rm -rf` は使わない

## Effort Level（考える深さ）の運用

- **普段はHigh（しっかり考える）で動きます**（`.claude/settings.local.json` の `effortLevel: "high"`）
- **もっと深く考えてほしい1問だけ**は、プロンプトに **`ultrathink`** と書くと最大思考（Max相当）で答えます
- **Pro契約のため、ultrathinkは本当に必要な時だけ使う**（使用上限の消費が早まるため）
- 詳しい運用・ヒアリング・オプション設定は `commerce/ebay/staff-ops/takahashi-effort-level-setup.md` を参照
- 使用上限がすぐ来ると感じたら社長に相談（MCP棚卸し等の対策あり）
