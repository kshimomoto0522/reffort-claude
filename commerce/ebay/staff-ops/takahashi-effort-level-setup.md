## 🎯 目的

高橋さんのClaude Codeを「普段はしっかり考える（High）、必要な時だけ最大思考（Max相当）」の状態に調整します。彼女はClaude Proプランを利用しており、使用上限が社長（Maxプラン）よりも厳しいため、**トークン消費を抑えつつ回答品質を保つ**ことが最優先です。

## 🧭 背景（高橋さんに簡単に説明してから進めてください）

- Claude Codeには4段階の思考深度があります：**low / medium / high / max**
- 最近、デフォルトが「medium（中くらい）」に変更されました
- 「medium」だと簡単な回答になりがちで、業務用途では物足りないことが多い
- **解決策：デフォルトを「high（しっかり考える）」に固定する**
- さらに深く考えてほしい1問だけは、プロンプトに **`ultrathink`** と書くと最大思考で答えます

## ✅ 基本セットアップ（全員共通・必ず実施）

### ステップ1：現在の設定ファイルを確認

```

このファイルが既にある場合は中身を読んでください。無い場合は、`commerce/ebay/staff-ops/takahashi-settings-template.json` の中身をコピーして作成されている前提です。

### ステップ2：effortLevel が "high" に設定されているか確認

**無い場合**：高橋さんに「このファイルに1行追加してほしいです」と手順を案内してください。彼女はプログラミング経験がないため、**Claude Codeから直接編集することは権限上できません**（`.claude/**` は deny 設定のため）。以下のように案内してください：

  "effortLevel": "high",

```

### ステップ3：設定後に動作を確認

再起動後、高橋さんに「試しに何か質問してみてください」と案内し、回答の質が向上していることを一緒に確認してください。

## 🚀 ultrathink の使い方（高橋さんに必ず教えてください）

### 基本

プロンプトに **`ultrathink`** という単語を入れるだけで、その1ターンだけ最大思考（Max相当）で答えます。

### 使用例

```
❌ これは普通のHigh思考：
   「このマニュアルを改善して」

✅ これは最大思考（ultrathink）：
   「このマニュアルを改善して。ultrathink」

✅ これも最大思考：
   「ultrathinkでこのマニュアルを根本から見直して提案してほしい」
```

### いつ使うべきか

| 場面 | ultrathinkを使う？ |
|---|---|
| 簡単な質問・確認 | ❌ 不要 |
| 文書の下書き作成 | ❌ 不要（Highで十分） |
| マニュアルの全体設計 | ✅ 使う |
| バグの原因調査 | ✅ 使う |
| 戦略・企画のブレスト | ✅ 使う |
| 「どちらが良いか」の比較検討 | ✅ 使う |

### ⚠️ Pro契約での注意

高橋さんはClaude **Pro**（月$20）契約です。社長のMax契約と違い、使用上限（5時間あたりのメッセージ数）が厳しめです。

- **ultrathinkは1回で通常の数倍〜10倍のトークンを使います**
- 多用すると使用上限にすぐ達します
- **本当に深く考えてほしい1問だけ**に使うのがコツです

## 🔧 オプション機能：キーワード自動ブースト（ヒアリング結果次第で判断）

高橋さんへのヒアリング（質問4）で **「『ちゃんと』『しっかり』『もっと』をよく使う」** という回答があった場合、社長の環境と同じように **特定キーワードを検出したら自動でultrathinkを発動する仕組み** も用意できます。

### ただしPro契約では慎重に

自動ブーストはトークン消費を早めます。以下のいずれかを高橋さんに選んでもらってください：

| 選択肢 | 動作 | Pro向け評価 |
|---|---|---|
| **A. 自動ブーストなし**（推奨） | `ultrathink` を自分で明示した時だけMax | ✅ 使用上限に優しい |
| **B. 限定自動ブースト** | 「しっかり」「ちゃんと」「もっと」で自動Max | ⚠️ 使用量が増える |
| **C. フル自動ブースト** | 複雑キーワード2個以上で自動Max（社長と同じ） | ❌ Pro契約では推奨しない |

**デフォルトは「A」を選んでおいて、2週間ほど運用してから高橋さんの実感で調整**するのがおすすめです。

### Bを選んだ場合の実装

Bを選ぶ場合は、**高橋さんのホームディレクトリに**（プロジェクト内の `.claude/` ではなく）フックスクリプトを配置します。以下の手順で進めてください：

1. `~/.claude/hooks/effort_booster_pro.py` を作成（以下のコードを使用）
2. `~/.claude/settings.json`（ユーザーレベル設定）にhook登録
3. テストして動作確認

**`effort_booster_pro.py` の中身（Pro向け保守版）：**

```python
# -*- coding: utf-8 -*-
"""
Claude Code Effort Auto-Booster Hook (Pro契約向け保守版)
高橋さん専用 - 明示的な品質強調語だけを拾う控えめ設定
"""
import sys
import io
import json
from datetime import datetime
from pathlib import Path

# Windows/Mac両対応のUTF-8固定
sys.stdin = io.TextIOWrapper(sys.stdin.buffer, encoding="utf-8", errors="replace")
sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding="utf-8", errors="replace")

# Pro契約向け：明示オプトイン語のみを拾う（自動判定はしない）
OPT_IN_KEYWORDS = [
    "!max", "ultrathink",
    "しっかり", "ちゃんと", "もっと", "じっくり", "徹底的",
]

LOG_PATH = Path(__file__).parent / "effort_booster_pro.log"

def log(msg: str) -> None:
    try:
        with LOG_PATH.open("a", encoding="utf-8") as f:
            f.write(f"{datetime.now().isoformat()} {msg}\n")
    except Exception:
        pass

def main() -> None:
    try:
        payload = json.load(sys.stdin)
    except Exception:
        sys.exit(0)
    prompt = payload.get("prompt", "") or ""
    for kw in OPT_IN_KEYWORDS:
        if kw.lower() in prompt.lower():
            log(f"boost: {kw}")
            print("ultrathink")
            sys.exit(0)
    log(f"skip (len={len(prompt)})")
    sys.exit(0)

if __name__ == "__main__":
    main()
```

**`~/.claude/settings.json` に追加する hook 設定：**

```json
{
  "hooks": {
    "UserPromptSubmit": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "python \"$HOME/.claude/hooks/effort_booster_pro.py\""
          }
        ]
      }
    ]
  }
}
```

※ Windowsの場合、`python` の代わりに `py` を使う必要がある場合があります。高橋さんの環境で `python --version` が通るかを確認してから設定してください。

## 🛟 使用上限に達してしまった場合の対処（高橋さんに伝えてください）

もし「使用上限がすぐ来てしまう」「反応が重い」と感じたら、以下の5つを順に試してもらってください：

1. **MCPの棚卸し** — 使っていない外部連携（MCP）をOFFにする
2. **MCPのSkill化** — たまにしか使わないMCPを Skill に変換（呼び出し時だけ読込）
3. **起動時読み込みファイルの圧縮** — CLAUDE.md や MEMORY.md を「目次＋詳細分離」構造にする
4. **プラグインの棚卸し** — 使っていないプラグインを削除
5. **`/context` コマンド** — 現状のトークン消費内訳を可視化

詳細は社長にお尋ねください。社長の環境では既に対策済みで、同じ方針を高橋さんの環境にも適用できます。

# 📚 補足資料：Effort Levelの4段階一覧

| レベル | 思考量 | 適した場面 |
|---|---|---|
| low | 最小 | 簡単な一覧取得・分類 |
| medium | 中（Claude Codeの新デフォルト） | 通常のコーディング |
| **high** | 大（← 今回設定する） | 複雑な分析・設計 |
| **max** | 最大（← ultrathinkで発動） | 難しい判断・経営レベルの思考 |
