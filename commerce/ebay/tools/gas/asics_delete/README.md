# 【ASICS】在庫管理 出品削除予約 GAS

## 概要

スプレッドシート上の Q列「出品削除」にチェックを入れた商品を、メニューから一括予約 → ASICSツールが実際の削除を実行する仕組み。

**設計**: GAS は予約のみ（軽量・即時完了）・実削除は Python ツール側

## ファイル

| ファイル | 役割 |
|---|---|
| `コード.js` | GAS本体（メニュー作成 + 予約処理） |
| `appsscript.json` | GASマニフェスト（V8 / Asia/Tokyo） |
| `.clasp.json` | claspデプロイ設定（scriptId 必須・社長作業後に更新） |

## セットアップ手順（一回限り）

### 1. GAS プロジェクト作成（社長作業）

1. 【ASICS】在庫管理 シートを開く
2. 上部メニュー：拡張機能 → Apps Script
3. 新規プロジェクト名を `【ASICS】出品削除予約GAS` 等に変更
4. URL から scriptId をコピー（`script.google.com/d/[scriptId]/edit` の部分）
5. Claude に scriptId を伝える

### 2. clasp 設定（Claude作業）

```bash
cd commerce/ebay/tools/gas/asics_delete
# .clasp.json の scriptId を社長から共有された値に書き換え
clasp push
```

### 3. メニュー有効化（社長作業）

1. シートをリロード（F5）
2. 上部に「ASICS出品管理」メニューが出現するか確認

### 4. 動作確認（社長作業）

1. テスト用に1件の商品の Q列にチェックを入れる
2. 「ASICS出品管理 → ✅ チェック商品を削除予約」を押す
3. 確認ダイアログ → OK
4. A1 に「出品削除予約済み...」と表示されるか確認
5. ASICSツールが起動中なら最大4分後に削除実行・完了メッセージが A1 に表示

## 仕様

### キュー方式

| セル | 役割 |
|---|---|
| `A1` | スタッフ向けステータス表示（"出品削除予約済み（X件...）" 等） |
| `Z1` | 隠しキューマーカー（"PENDING" or 空） |
| `Q列` | 各商品のチェックボックス |

### 動作フロー

```
[GAS]
  ボタン押下 → 確認ダイアログ → OK
  → A1 にメッセージ表示
  → Z1 = "PENDING"
  → 完了

[Python ツール]
  各アイテム処理直前 + 休憩中（30秒ごと）に Z1 をポーリング
  → "PENDING" 検知
  → Q列TRUE の行を抽出
  → eBay EndItem API 呼び出し
    成功: 行のA〜S列を全クリア
    Policy違反: R列に「Policy違反の為要手動削除」
    その他失敗: R列にエラー詳細
  → Z1 をクリア
  → 通常処理に戻る
```

## 編集後のデプロイ

```bash
cd commerce/ebay/tools/gas/asics_delete
clasp push
```

## トラブルシューティング

### 「ASICS出品管理」メニューが出ない

- シートをリロード
- それでも出ない場合: GAS エディタで `onOpen` を1回手動実行（実行 → 関数を選択 → onOpen）

### A1 に予約メッセージが出ない

- GAS エディタで Apps Script 実行ログを確認
- スクリプトが該当シート名「【ASICS】在庫管理」に紐づいているか確認

### Z1 を手動でクリアしたい

- スプレッドシート上で Z1 セルを選択 → Delete キー
- Pythonツール側のキュー処理がスキップされる
