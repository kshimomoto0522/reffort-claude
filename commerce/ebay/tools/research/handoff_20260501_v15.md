# eBay 無在庫リサーチツール Ver.1.5 ハンドオフ（2026-05-01）

## 社長指摘の Ver.1 の重大欠陥

社長から朝（2026-05-01）に厳しい指摘を受け、Ver.1 の以下の根本的な問題を全部認めて改修した：

1. **新品リサーチなのに中古商品を仕入候補にした**
   - 例: Air Jordan 1 $100 → 楽天「【中古】¥12,760」を採用して利益 $10 と表示
   - これは出品したらクレーム不可避
2. **スニーカー過度な比率（14 中 6 銘柄＝43%）**
   - Reffort のコア事業に引きずられて広く調べていない
3. **赤字 16 件をレポートに混入**
4. **仕入先 2 サイトのみ（楽天・Yahoo!ショッピング）で「最安発見率」が低い**
5. **「売れる根拠」が一切ない**
   - watch 数・販売数・競合数・価格分布なし
   - 「これ本当に売れるの？」と聞かれて答えられない状態

## Ver.1.5 で何を直したか

| Ver.1 の問題 | Ver.1.5 の対処 |
|---|---|
| 中古商品混入 | 楽天/Yahoo/Amazon/フリマ/駿河屋の全サプライヤに `condition` 判定（`【中古】`等のマーカー除外） |
| スニーカー偏重 | キーワードを 12 カテゴリ全網羅に拡張（JDM 自動車パーツ・トレカ・カメラ・腕時計・楽器・釣具・レトロゲーム・万年筆・包丁・ヴィンテージ古着・コレクター系・スニーカー縮小） |
| 赤字混入 | ゲート機能：純利益≥$15、マージン≥15%、根拠スコア≥45 点を満たす候補のみ HTML 出力 |
| 仕入先 2 サイトのみ | **5 サイト並列**：楽天・Yahoo!ショッピング・**Amazon.co.jp**（curl_cffi + chrome131 TLS 偽装）・**Yahoo!フリマ**（内部 JSON API）・**駿河屋**（コレクター系） |
| 売れる根拠なし | **7 シグナル合成スコア（0-100）**：watch 数・競合数・同型番セラー数・出品総数・価格競争力・出品長寿性・上位セラー実績 |

## 売れる根拠スコアの設計

| シグナル | 重み | 取得元 |
|---------|------|--------|
| watch 数 | 25 点 | Browse API restricted（要 App Check 申請）・未取得時は他 6 シグナルに按分 |
| 競合セラー数 | 20 点 | Browse API レスポンスから自前集計（釣鐘型評価・8-25 で最大点） |
| 同型番セラー数 | 15 点 | matcher.py の型番抽出 + サンプル検索の合致集計 |
| 出品総数 | 10 点 | Browse API search response の `total` |
| 価格競争力 | 15 点 | 自分の想定売価 vs 中央値（0.92-0.98 倍で最大点） |
| 出品の長寿性 | 10 点 | itemCreationDate から最古日数算出 |
| 上位セラーの evals | 5 点 | サンプル中 max(feedbackScore) |

→ **75 点以上 = 強い根拠 / 60-74 = 中 / 45-59 = 弱い / 45 未満 = 不十分（候補から除外）**

## 限界の正直な明示

このツールは「**Browse API のアクティブ出品ベース**」で「売れる根拠」を組み立てている。
理想は eBay Marketplace Insights API（過去 90 日 SOLD データ）だが、申請承認制で受付が止まっている報告あり。

その代わり、以下の論理で「売れる根拠」を構築している：
- 「複数セラーが同じ型番を出品中 = その商品は売れている確証」（社長指摘から派生）
- 「市場全体の出品数が多い = 需要がある」
- 「日本セラーが複数いる = Reffort と同じビジネスモデルで売っている確証」
- 「価格中央値より安く出せる = 競合に勝てる」

これは「過去の実販売実績」より弱いが、**現在進行形の競合状況**を捉えている分、今この瞬間の判断材料としては相当強い。

ただし、**最終的に「これは出品して売れる」と確信を持つには、Reffort の自社実績や Marketplace Insights のデータが必要**。Ver.2 でそこまで持ち上げる。
