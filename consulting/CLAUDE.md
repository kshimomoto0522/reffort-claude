## 事業コンセプト

### 何をする事業か
下元 敬介氏が実際に「Claude Code × eBay運営」を実践・改善していく過程そのものをコンテンツ化し、
将来的にコミュニティ（スクール・コンサル）として提供する事業。

**核心価値：「実際にやっている人が、やりながら教える」**

- 理論ではなく「自分が実際にやって効果が出た方法」を届ける
- 失敗・試行錯誤・改善プロセスもすべてコンテンツになる
- eBayセラーの中でAI活用は圧倒的に差別化になる（ほぼ誰もやっていない）

## カリキュラム構想（現時点のアイデア）

### Unit 1：Claude Code入門
- Claude Codeとは何か（AIツールの中での位置づけ）
- インストール・初期設定・CLAUDE.mdの作り方
- 基本的な使い方：ファイル読み書き・質問・指示の出し方
- セッション管理・メモリの仕組み
- MCPとは何か・何ができるか

### Unit 2：eBay × Claude Code の基礎
- eBayのデータ構造を理解する（レポートの種類・項目の意味）
- レポートのダウンロード → Claudeに分析させる
- 週次チェックリストをClaudeと一緒に作る
- Chatworkへの自動通知を設定する

### Unit 3：eBayパフォーマンス分析
- CVR・CTR・Impressionsの読み方と改善アクション
- 商品の4分類（削除・育成・準売れ筋・コア）
- 広告（PLG・PLP・Offsite）の仕組みと最適化の考え方
- 週次レポートを自動生成する仕組みを作る

### Unit 4：業務自動化・効率化
- Pythonスクリプトを使った分析自動化（コード不要・Claudeが書く）
- eBay APIの活用（データ取得の自動化）
- Chatworkやその他ツールとの連携（MCP活用）

### Unit 5：CS対応 × AI（BayChat的な活用）
- バイヤーへの返信をAIに生成させる
- クレーム・返品対応のテンプレート化
- 多言語対応（日英翻訳）

### Unit 6：経営視点でのAI活用
- 収支・KPIをAIで可視化する
- 意思決定をAIにサポートさせる（何を売るか・何を止めるか）
- AI活用ロードマップの作り方（自分の事業への適用）

## Claudeへの役割（このフォルダ専用）

1. **日々の実践ログを記録する**
   - 下元氏がeClaude Codeで実施したこと・学んだことを`journey-log.md`に蓄積
   - セッションをまたいで継続的に更新する

2. **コンテンツ化の提案**
   - 「この作業、Unit 3のネタになりますね」などと随時提案する
   - 学習素材として使えるポイントを見逃さない

3. **カリキュラム設計のサポート**
   - 蓄積されたログをもとに、学習順序・難易度・構成を整理する

4. **教材・テンプレートの作成**
   - 指示があればユニットごとのテキスト・手順書・テンプレートを生成する

---

## Claudeへの重要ルール（コンサル部門専用）

- 下元氏の実体験・実数値は最高のコンテンツ素材。感じたこと・迷ったことも記録する
- 「将来教える前提」で常に「なぜそうするのか」の理由まで記録する
- 失敗・試行錯誤・遠回りしたことも正直に記録する（これがリアルなコンテンツになる）
- eBayの数値・API・広告等の具体的な知識も、初心者向けに説明できる形で残す
- BayChat・BayPackはReffort自社サービスのため、カリキュラムには含めない
