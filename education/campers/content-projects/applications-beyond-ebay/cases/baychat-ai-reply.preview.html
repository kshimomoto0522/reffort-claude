何をやったか（目的・背景）

Reffort 自社SaaS「BayChat」（eBayセラー向け顧客対応サービス）の AI Reply 機能（バイヤーへの返信を AI で自動生成）の開発・改善プロジェクト。

社長は AI Reply のプロンプト設計・テスト・本番反映を Claude Code で進めている。Cowatech（ベトナム開発会社）への発注も含めて、全体設計図の整備プロジェクトに発展。

Campersメンバーへの伝え方（参考例として）

ストーリー

「私の場合、自社サービス（BayChat）にAI返信機能を載せる開発も Claude Code でやっています。プロンプトのテスト・改善サイクルを Claude と一緒に回しながら、外注先への発注もすべて Claude が下書きを作ってくれる」

何が示せるか

  • eBay運営だけでなく、自社SaaSの開発もClaude Code で支援できる
  • プロンプト設計のような「AI を使うAI開発」も可能
  • 外注先（ベトナムのCowatech）への発注ドキュメント作成も Claude が支援

メンバーへの含意

  • 「皆さんも eBay以外の事業や新規事業を考えるとき、Claude Code が同じように支援できる」
  • 「業務AI化は eBay に限定されない」
  • ただしこの事例自体を真似する必要はない（メンバーは自社SaaSを持っていない）

つまづいた点（参考エピソード）

  • ハードコード定数の罠：3テナント分の値をコード内に直書きしていて重大バグの原因に
  • オフラインテスト ≠ 本番：本番ペイロード解析で +2pt 改善
  • 議論の結論を草案に反映する責任：会議で決まったことを後から忘れる
  • Slack送信ルール違反3重：確認なし送信を3回もやってしまった

ウェビナーでの紹介ポイント

「自分はASICSの在庫管理だけじゃなく、自社サービスの中でも Claude Code を使っている。プロンプトの改善サイクルも、外注先への発注書作成も、テストフレームワークも全部 Claude と一緒に作った」

→ 「皆さんも、eBay 運営の AI化が一段落したら、別の事業や新規事業で Claude Code を活用する方向にも目が向くはず」
