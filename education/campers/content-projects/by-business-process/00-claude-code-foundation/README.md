# 00. Claude Code 基盤 — メンバー向け導入・運用ガイド

> このフォルダは **Campersメンバーが Claude Code を導入し、自分の業務をAI化する第一歩を踏み出すための基幹資料**です。
>
> 質問する前にまず本資料を読み、最低限の導入を自分で実行してください。

---

## なぜ「Claude Code 基盤」が最初に必要か

Reffort社長が約7週間（2026-03-11〜04-28）試行錯誤してたどり着いた**運用ルール・注意事項・セキュリティ設定**は、メンバーが同じ遠回りをせずに済むための**地図**になります。

この基盤がしっかりしていれば、その後の業務AI化（リサーチ・出品・在庫・仕入・経理・分析・広告 等）はすべて、ここを土台に積み上げられます。

逆にこの土台がガタガタだと：
- セキュリティ事故（APIトークン漏洩等）
- Claude Code が肥大化して動作が重くなる・誤った回答を返す
- セッションが進化しない（メモリ機能を使えていない）
- 失敗を繰り返す
…という事態になります。

## 推奨される進め方（メンバー視点）

### ステップ1：環境準備（30分〜1時間）
1. `01_setup-installation.md` を読んで Claude Code をインストール
2. `02_security-rules.md` を読んで .env管理を理解
3. APIキー類を発行・.envファイルにセット

### ステップ2：プロジェクト立ち上げ（1〜2時間）
4. `03_claude-md-template.md` をコピーして自分のフォルダに `CLAUDE.md` として配置
5. テンプレートの空欄（自分の事業情報・取扱商材・目標 等）を埋める
6. Claude Code を起動して動作確認

### ステップ3：継続運用（毎セッション）
7. `04_session-and-memory.md` でセッション・メモリの使い方を理解
8. `05_dos-and-donts.md` の禁止事項を頭に入れる
9. 業務段階別フォルダ（01-research〜11-direct-sales）を順次活用

### ステップ4：定期メンテ（隔週）
10. `06_maintenance-cycle.md` の肥大化対策を実施
11. `07_complete-setup-checklist.md` で抜け漏れを定期確認

## 社長の試行錯誤の結晶（このガイドに含まれる主な学び）

- **APIトークンは絶対に直書きしない**（.env管理徹底・コード上には書かない）
- **CLAUDE.md は200行以下に抑える**（肥大化はAIをバカにする）
- **archive/フォルダで完了情報を退避**（情報資産保全とコンテキスト軽量化を両立）
- **隔週メンテで肥大化を物理的に防ぐ**
- **同じ指示を3回以上やり直していたら設計が間違っている**
- **`.env`は3原則：自動特定／自動オープン／開いた事実を先に宣言**
