# CLAUDE.md テンプレート — メンバーがコピペして使える初期設定

> 対象: Campersメンバー
> 使い方: 以下の枠内をコピーして自分のプロジェクトフォルダのトップに `CLAUDE.md` という名前で保存。空欄を埋める。

---

## CLAUDE.md とは

Claude Code はプロジェクトフォルダにある `CLAUDE.md` を**毎セッション最初に必ず読み込みます**。
ここに書いた内容が「あなたのClaudeの基本方針」になります。

社長が試行錯誤の末に確立した構造をテンプレート化しました。

---

## テンプレート（コピペ用）

以下を `CLAUDE.md` にコピーして、`【】`内を自分の事業情報で埋めてください。

```markdown
# 【あなたの事業名】 — Claude Code コアルール

> 毎セッション最優先・本ファイルは80行以下に保つ（肥大化防止）

---

## 私について（要点）

- 名前: 【あなたの名前】
- 事業: 【eBay輸出 / メルカリ / その他フリマ等】
- 取扱商材: 【スニーカー / アパレル / 雑貨 / 他】
- プログラミング経験: 【未経験 / 初心者 / 中級】
- 業務スタッフ: 【外注○名 / 自分のみ】
- 月間売上規模: 【記入任意】
- 1年の目標: 【例：AI化で業務時間半減・売上維持／関税対応で利益率回復／等】

---

## 重要ルール

- APIトークンは必ず `.env` 管理（コードに直書き禁止）
- 本番の顧客／注文データを操作する作業は必ず私（オーナー）の確認を得る
- 同じ指示を3回以上やり直していたら設計が間違っている合図
- セッションが重くなったら新セッションへの引継ぎを提案
- 出来ない事は素直に「出来ない」と言う（嘘をついて完了報告しない）
- 開ける／確認できる／実行できる事は私に依頼せず Claude が自分でやる

---

## 私の業務プロセス（AI化したい順に並べる）

優先度を1〜5で記入：

| 業務 | AI化優先度 | メモ |

| リサーチ | 【】 | 【現状の手間・課題】 |
| 出品 | 【】 | 【】 |
| 在庫管理 | 【】 | 【】 |
| 仕入管理 | 【】 | 【】 |
| メッセージ対応 | 【】 | 【】 |
| 発送 | 【】 | 【】 |
| 経理・確定申告 | 【】 | 【】 |
| 分析・KPI | 【】 | 【】 |
| 広告 | 【】 | 【】 |

---

## 回答スタイル

- 専門用語は必ず簡単な説明を添える・素人向けに
- 選択肢提示→私が判断
- 数字は「だから何をすべきか」まで踏み込む
- 私の判断が最適でないと感じた時は理由添えて軌道修正提案

---

## セッション終了時のチェックリスト

「終了」「おつかれ」等の終了示唆発言で実行：

1. 今日やったことを `journey-log.md` に追記
2. 重要な意思決定は memory に保存
3. 設定変更があれば本ファイルに反映
```

## カスタマイズのポイント

### 1. 「私について（要点）」は具体的に
- Claude は毎回これを読むので、毎回同じ説明をしなくて済む
- 「未経験」と書けば自動で平易な日本語で答えてくれる
- 取扱商材を書けば、関連する話題で先回り提案してくれる

### 2. 「業務プロセスの優先度」を埋める
- 自分の業務でどこから着手すべきかが明確になる
- 優先度5（即必要）から順にAI化していくロードマップになる
- これが空欄だと「何から始めるか」が曖昧で時間を浪費する

### 3. 「回答スタイル」は妥協しない
- メンバー本人がプログラミング未経験なら「素人向け」を絶対書く
- これがないとClaudeは技術用語連発で説明する

### 4. 80行以下を維持する
- 社長が試行錯誤して確立したルール
- これを超えると「肥大化」が始まり、Claude のレスポンスが悪化する
- 詳細は `06_maintenance-cycle.md` を参照

## 応用：部門別 CLAUDE.md

複数の業務がある場合は、フォルダごとに CLAUDE.md を置けます：

```
my-ebay-business/
  CLAUDE.md                    ← トップ：基本方針
  research/
    CLAUDE.md                  ← リサーチ業務専用
  shipping/
    CLAUDE.md                  ← 発送業務専用
```

Claude Code は**作業中のフォルダから親方向にCLAUDE.mdを遡って読み込みます**。
research/ で作業すると、`my-ebay-business/CLAUDE.md` + `research/CLAUDE.md` の両方が自動ロードされます。

これは応用なので、最初はトップに1つだけ置いて運用してください。
