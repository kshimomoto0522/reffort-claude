セッション管理・メモリ仕組み

対象: Campersメンバー 所要時間: 15〜20分の理解＋実践

このファイルでわかること

  • Claude Code の「セッション」とは何か
  • 「メモリ」がどう動くか
  • セッションが重くなったときの対処法
  • 過去の会話を引き継ぐ方法

セッションとは

Claude Code を起動して、終了するまでの一連の会話 = 1セッション です。

claude       # ← セッション開始
> ...        # ← 会話・作業
> /exit      # ← セッション終了
  • 1セッション中の会話は連続している（前の発言を覚えている）
  • セッションを終了すると、その会話は記録には残るが「現在の文脈」からは消える
  • 次のセッションは「白紙」から始まる

ただしメモリ機能を使えば、セッションをまたいで覚えてくれます。

メモリ仕組み

Claude Code には3種類の「記憶」があります：

1. CLAUDE.md（毎セッション読み込まれる）

プロジェクトフォルダのCLAUDE.mdは毎セッション最初に必ず読まれます。 ここに書いた内容 = Claudeの基本方針。

2. memory/フォルダ（明示的に保存・参照する記憶）

過去の決定・フィードバック・重要な情報を保存しておく場所。 Claudeが必要に応じて読み込んで判断材料にします。

社長の例： - memory/user_profile.md — 社長のプロフィール（毎回読む） - memory/feedback_security.md — セキュリティルール - memory/project_consulting.md — コンサル事業の方針

3. journey-log.md（時系列の業務記録）

日々の作業・学び・失敗を時系列で蓄積するファイル。 コンテンツ素材としても、過去の参照記録としても価値が高い。

セッションが重くなったときの対処法

長時間作業していると、Claudeのレスポンスが遅く・回答が浅くなります。 これは コンテキスト（会話履歴）が膨らみすぎた 状態。

対処1：新セッションを開始

/exit              # 一度終了
claude             # 再起動

重要な情報はCLAUDE.mdとmemory/に保存されているので、引き継ぎはほぼ自動。

対処2：明示的にメモリを保存

セッション終了前に：

今日の重要な学びをmemoryに保存して
今日やったことをjourney-log.mdに追記して

対処3：CLAUDE.mdの肥大化を点検

セッションが頻繁に重くなるなら、CLAUDE.mdが200行を超えている可能性。 詳細は 06_maintenance-cycle.md を参照。

過去の会話を引き継ぐ方法

パターン1：自然な引継ぎ（メモリ使用）

  1. 重要な決定をしたら「memory に保存して」と頼む
  2. 次セッションでClaude が自動で読み込んで認識

パターン2：明示的な引継ぎ（handoff 文書）

大きなプロジェクトの途中で session を切るときは、引継ぎドキュメントを作る：

このプロジェクトの引継ぎ文書を `.claude/handoff_<日付>_<テーマ>.md` に作って。
次セッションで `@<その文書> を読んで進めて` と言えば再開できる形で。

パターン3：journey-log.md 経由

セッション終了時に必ず journey-log に追記。 次セッションで「journey-log を見て今日何やる？」と聞けば、Claudeが過去を踏まえて提案。

メモリの種類（社長運用に倣う）

memory/ 配下に保存するときは、種類を意識すると後で取り出しやすい：

種類 内容 例
user あなた自身のプロフィール・スキル user_profile.md
feedback Claude への指示・修正方針 feedback_communication.md
project 進行中のプロジェクト情報 project_ad_optimization.md
reference 外部リソースへの参照 reference_ebay_api.md

社長は約30〜40個のmemoryファイルで運用しています（多すぎると逆効果）。

やってはいけないこと

NG1：メモリに何でも書きすぎる

  • 50ファイル超えると Claude が把握しきれない
  • 重要なものだけ・1ファイル100行以下で運用

NG2：CLAUDE.md にメモリの内容を全部書く

  • CLAUDE.mdは200行以下に保つ
  • 詳細はmemory/に分割

NG3：journey-log を整形しすぎて書かなくなる

  • 完璧な書式より「とにかく書く」
  • 後からClaudeに整形させればOK
