運用維持の隔週メンテ・肥大化対策

対象: Campersメンバー（Claude Code 運用が3ヶ月以上経過した人） 所要時間: 隔週30分〜1時間

なぜ「メンテ」が必要か

Claude Code は使うほどファイル・設定・記録が溜まっていきます。

放置していると： - CLAUDE.md が500行を超えて肥大化 - memory が50個超えて Claude が把握できない - settings.local.json の権限が古い・危険なまま - archive すべき完了情報が現役フォルダに混在 - 結果：Claude が「バカ」になる（社長体験：2026-04-23）

逆に隔週で点検する習慣を持てば： - 常にClaudeが鋭い回答を返す - セキュリティが古いまま放置されない - 業務AI化のスピードが落ちない

肥大化のサイン（こんな症状が出たら点検時）

症状 何が起きているか
同じ指示を3回以上やり直している コンテキスト混乱・指示が伝わりきっていない
Claude の回答が遅い コンテキストが膨らみすぎ
Claude が「確認しました」と嘘をつく 注意力低下・コンテキスト圧迫
トークン消費が跳ね上がる 不要ファイルを読み込みすぎ
「あれ、前にも同じ間違いした」 学習が記録されていない

隔週メンテのチェック項目（30分〜1時間）

チェック1：CLAUDE.md の行数

wc -l CLAUDE.md
結果 判定
〜80行 ◎ 理想
81〜150行 ○ OK
151〜200行 △ 整理推奨
201行〜 ✕ 即整理（ファイル分割）

整理方法: 詳細を別ファイル（memory/ や .claude/rules/）に分けて、CLAUDE.md は要点のみに絞る。

チェック2：memory ファイル数

ls memory/ | wc -l
結果 判定
〜30個 ◎
31〜50個 ○
51個〜 △ 統合推奨

統合方法: 似た内容のファイルを1つにまとめる、古い情報は archive に退避。

チェック3：settings.local.json の権限見直し

.claude/settings.local.json を開いて allow/deny リストを確認： - 使っていないコマンドは削除 - ワイルドカード（Bash(*) 等）は危険なので個別化 - 怪しいコマンドが残っていないか

チェック4：archive フォルダの活用

完了したプロジェクト・古いツールは archive/ 配下に移動： - 削除ではなく退避（情報資産は保全） - Claude は archive/ を読まないので軽量化される

チェック5：journey-log.md の追記漏れ

直近2週間で書いていない日があれば、今からでも追記： - 「何かやったはず」を絞り出してメモ - 後でコンテンツ素材になる

社長の隔週メンテ運用（参考）

社長は隔週メンテを完全自動化しています：

自動チェック（毎月第1・第3月曜10:00）

  • 計測スクリプトが定量チェック
  • Chatwork個人DMに結果通知
  • 「CLAUDE.md 200行超えています」「memory 50個超えています」等の警告

半自動改善（社長判断）

  • /隔週メンテナンス スラッシュコマンドで以下を一気に：
  • Chatwork DM 読み込み
  • 5層調査（CLAUDE.md／memory／settings／hook／archive）
  • 松竹梅の改善提案
  • 社長判断 → 実行 → 結果再送信

結果蓄積

  • 各メンテサイクルの過程を archived-by-theme/claude-code-maintenance-case-study/session-logs/ に保存
  • コンテンツ素材としても活用

メンバー向けの簡易メンテ手順（最初はこれから）

ステップ1：ターミナルで以下を実行

wc -l CLAUDE.md
ls memory/ | wc -l
git status

ステップ2：Claude に投げる

隔週メンテをお願いします。
- CLAUDE.md の肥大化チェック
- memory の整理候補
- 不要ファイルの archive 移動候補
を教えてください。

ステップ3：提案された改善を1つずつ実施

  • 全部一気にやろうとしない
  • 1つ実施・動作確認・次へ

ステップ4：journey-log.md に追記

2026-XX-XX 隔週メンテ実施
- CLAUDE.md: XXX行 → XXX行
- memory: XX個 → XX個
- 改善: <具体的に>

やってはいけないメンテ作業

NG1：いきなり大量削除

  • 必ずバックアップ（git commit）してから
  • 削除より archive 退避を優先

NG2：「とりあえず全部新しくする」

  • 過去の記録は財産
  • 整理 ≠ リセット

NG3：メンテを後回しにする

  • 「忙しいから来週」が3ヶ月続く
  • 隔週で自動アラート（カレンダー登録 or scheduled task）

社長が肥大化を放置した結果（実例）

2026-04-23、社長から：

「最近あきらかにClaude codeが一度言った通りにやらない、嘘をつく、言い訳をする、遅い、トークン消費が激しい、回答がバカ過ぎます」

調査の結果： - ルートCLAUDE.md 136行 - 最大子CLAUDE.md 488行（推奨200行の2.4倍） - memory 47ファイル・2,385行 - settings.local.json 298行 - 不要ファイル 77個・190MB

→ 大改修（梅案＋竹案）を実施し、すべて軽量化・回答品質が劇的回復。
