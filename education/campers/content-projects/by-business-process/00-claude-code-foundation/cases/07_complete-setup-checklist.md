# 導入完了チェックリスト

> 対象: Claude Code 導入を完了したCampersメンバー
> 使い方: 各項目に「☐→☑」をつけながら進める

---

## 環境構築（基礎）

- ☐ Node.js LTS版がインストール済み
- ☐ Claude Code がインストール済み（`claude --version` で確認）
- ☐ Claude Max プランに加入済み
- ☐ ターミナルから `claude` コマンドで起動できる
- ☐ ブラウザ認証が完了している
- ☐ Git がインストール済み
- ☐ GitHubアカウント を持っている（Privateリポジトリ作成可能）

参照: `01_setup-installation.md`

---

## セキュリティ初期設定

- ☐ プロジェクトフォルダのトップに `.env` を作成済み
- ☐ `.gitignore` に `.env` が含まれている
- ☐ APIトークンは全て `.env` に格納（コードに直書きゼロ）
- ☐ 過去にコードに書いたAPIトークンがあれば再発行済み
- ☐ GitHub のリポジトリは Private 設定
- ☐ 自動バックアップ運用を決めた（毎日 / 週次 / 都度）
- ☐ Claude Code に `.env` の中身を表示させない運用ルールを理解

参照: `02_security-rules.md`

---

## CLAUDE.md 整備

- ☐ プロジェクトフォルダのトップに `CLAUDE.md` を作成済み
- ☐ 「私について」を埋めた（事業・商材・経験・目標）
- ☐ 「業務プロセス」の優先度を埋めた
- ☐ 「回答スタイル」を自分のレベルに合わせて記入
- ☐ 80行以下に収まっている
- ☐ 重要ルール・セッション終了チェックリストを記入

参照: `03_claude-md-template.md`

---

## メモリ・記録

- ☐ memory/ フォルダを作成済み
- ☐ `memory/user_profile.md` で自己プロフィールを書いた
- ☐ `journey-log.md` を作成済み
- ☐ 初回セッション後に journey-log を1エントリ書いた
- ☐ 次セッションで journey-log が引き継がれることを確認

参照: `04_session-and-memory.md`

---

## DO・DON'T の理解

- ☐ APIトークンを直書きしてはいけない理由を説明できる
- ☐ 本番データを触る前に必ず確認する習慣がある
- ☐ Claude が「分からない」と言ったら信じる
- ☐ 失敗を隠さず journey-log に記録する習慣がある
- ☐ 同じ指示を3回やり直したら設計が間違っていると気付ける

参照: `05_dos-and-donts.md`
