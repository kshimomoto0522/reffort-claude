# 01. リサーチ — 売れる商品を見つける業務のAI化

> このフォルダは「売れる商品を探す」リサーチ業務をAI化する事例を集める棚です。

---

## この業務段階で何ができるか（メンバー向け）

### 課題（メンバーが抱えがちな悩み）
- リサーチに時間がかかる（毎日数時間〜半日）
- 売れ筋を見つけるための「目」が属人化している
- 競合セラーの動きを追いきれない
- 複数サイト（メルカリ／Yahoo!オク／Amazon／海外サイト）の横断調査がしんどい

### Claude Code でできること
- **競合セラーの出品変化を自動追跡**（毎日の差分を Chatwork に通知）
- **eBayの商品検索を API で自動化**（売れ筋カテゴリの動向を毎朝レポート）
- **Excel/CSVで管理しているリサーチ候補リストの自動精査**（HIROUNエクセル等）
- **メルカリ・Yahooオクの商品ページから情報抽出**（商品名・価格・状態・出品者）
- **「売れている商品の特徴」をデータから抽出**（タイトル傾向・価格帯・写真パターン）

## メンバーが最初に試すべきこと

1. **自分のリサーチ業務を棚卸し**（毎日何時間使っているか・どのサイトを見るか）
2. **最も時間がかかる作業を1つ特定**
3. それをClaude Codeで自動化できないか相談
   - 例：「競合セラーAの出品リストを毎朝チェックして、新規追加・価格変更を Chatwork に通知してほしい」
   - 例：「メルカリで○○というキーワードの新規出品を1時間ごとにチェックしてほしい」

## つまづきポイント（社長の体験から）

- **スクレイピングは Bot 検出との戦い**（詳細：03-mukozai-stock-management の cases/stock-management-tool.md）
- **API公開がない場合は工夫が必要**（公式API → 非公式スクレイピング → 手動の優先順位）
- **完璧を目指さない**（80%取れれば十分・100%取ろうとして詰まらない）
