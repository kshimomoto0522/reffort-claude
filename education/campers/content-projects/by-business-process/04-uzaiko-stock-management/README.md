# 04. 有在庫管理ツール — 自社在庫追跡のAI化

> このフォルダは「自社在庫を抱えて販売する」有在庫モデルの在庫管理AI化事例を集める棚です。

---

## この業務段階で何ができるか（メンバー向け）

### 課題（メンバーが抱えがちな悩み）
- 在庫数の把握をスプレッドシートで頑張っているがミスが多い
- 倉庫の物理在庫と帳簿が一致しない
- 在庫回転率が分からず「眠っている在庫」が見えない
- 複数チャネル（eBay／自社サイト／実店舗）の在庫を一元管理できない
- 入荷・出荷のタイミングで人為ミス

### Claude Code でできること
- **スプレッドシート在庫表の自動メンテナンス**（GAS / Google Sheets API）
- **入荷・出荷を eBay 受注データから自動反映**（API連携）
- **在庫回転率レポート自動化**（眠っている在庫を週次でアラート）
- **複数チャネル在庫の一元管理**（自社サイト・eBay・倉庫を横断）
- **発注タイミング自動アラート**（「あと10個切ったら発注」）

## メンバーが最初に試すべきこと

### ステップ1：現状の在庫管理を棚卸し
- スプレッドシートで管理している？それとも Excel？紙？
- 在庫を更新するタイミングは？（入荷時・出荷時・週次・月次）
- 複数チャネルがある場合、どう一元管理している？

### ステップ2：ボトルネックを特定
- 入荷時の登録が漏れる？
- 出荷時の在庫減が反映されない？
- 倉庫実在庫と帳簿のズレ？

### ステップ3：Claude に相談
- 例：「スプレッドシートの在庫管理表に、eBay受注APIから出荷分を自動反映させたい」
- 例：「入荷予定をフォーム入力したら自動で在庫表に反映する仕組みを作りたい」

## つまづきポイント（社長の経験予想）

- **既存スプレッドシートの構造が複雑だと自動化が困難**（最初に整理が必要）
- **複数チャネル横断は API キーが多くて管理が大変**（.env管理徹底）
- **倉庫実在庫の差異**（システム化しても物理ミスは防げない・棚卸しは必須）

---

## メンバーへの含意

- **完全自動化より「ミス検知」のほうが現実的**（人間がチェックする・Claude が異常を見つける）
- **スプレッドシートをやめる必要はない**（Sheets API で連携可能・既存運用を活かす）
- **eBayと自社サイトの両方やる人は特にメリット大**（手動の二重入力が消える）
