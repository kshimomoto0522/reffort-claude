# 06. トラブル対応 — クレーム・返品・問題発生時のAI化（要検討）

> このフォルダは「トラブル対応」業務のAI化事例を集める棚です。
>
> ⚠️ **このセクションは要検討です**（社長判断）。AI化できる範囲が限定的で、Claude Code を使う優位性が他段階より低いため、コンテンツ化の優先度は低めです。一旦枠だけ作り、今後の蓄積を見て判断します。

---

## この業務段階で何ができるか（メンバー向け・現時点の想定）

### 課題（メンバーが抱えがちな悩み）
- クレームが来たとき、過去事例を思い出せない
- 返品の判断基準がスタッフごとにバラバラ
- 重大トラブル（取引中止・eBay制裁等）の対応が属人化
- 同じトラブルが繰り返し発生する

### Claude Code でできること（限定的）
- **過去のトラブル対応記録の検索・参照**（journey-log や個別ノートから）
- **トラブル分類の自動化**（受信メッセージから「これは返品案件」「これは配送遅延」）
- **対応テンプレの自動生成**（メッセージ対応は BayChat マターのため、Claude Code は使わない方針）
- **トラブル発生時の根本原因分析**（複数事例を横断してパターン抽出）

### 制約
- メッセージ対応そのものは BayChat に統一（Claude Code で AI Reply を作るのは推奨しない）
- 物理対応（返品受領・検品・再送）は人間の作業
- → AI化できる範囲は「分析・参照・分類」に限定される
