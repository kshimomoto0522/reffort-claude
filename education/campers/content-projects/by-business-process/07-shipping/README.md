# 07. 発送 — 国際発送業務のAI化

> このフォルダは「eBayで売れた商品を発送する」業務のAI化事例を集める棚です。

---

## この業務段階で何ができるか（メンバー向け）

### 課題（メンバーが抱えがちな悩み）
- 発送期限（Ship by Date）の管理がスタッフ任せでミスが起きる
- 配送方法・送料計算が複雑
- 追跡番号の eBay 反映が手動
- 発送遅延でDSR（評価）に悪影響
- 倉庫スタッフへの発送指示が口頭・属人化

### Claude Code でできること
- **発送期限の自動管理**（Ship by Date を仕入管理表で可視化・前日アラート）
- **発送指示書の自動生成**（受注情報 → ピッキングリスト → 倉庫スタッフへ配信）
- **追跡番号の自動 eBay 反映**（OAuth2 + Sell API）
- **発送遅延の事前検知**（リスク高い案件を Chatwork に通知）
- **配送方法の自動判定**（重量・サイズ・配送先で最適便を提案）

## メンバーが最初に試すべきこと

### ステップ1：現状の発送フローを棚卸し
- 受注 → 仕入指示 → 入荷 → 検品 → 発送 → 追跡反映 のどこで時間取られている？
- ミスが起きやすいのはどこ？（発送期限見落とし？追跡番号貼り間違い？）

### ステップ2：自動化候補を選ぶ
- 発送期限管理：仕入管理表と連携（05-procurement 参照）
- 発送指示書：受注情報からピッキングリスト自動生成
- 追跡番号反映：API で一括反映

### ステップ3：Claude に相談
- 例：「明日が Ship by Date の注文を毎朝 Chatwork に通知して」
- 例：「追跡番号入力したら自動で eBay に反映する仕組み作って」

## 社長の体験（仕入管理表との連携）

社長の運用では、**仕入管理表（GAS）が発送管理も兼ねている**：
- eBay受注 → 仕入指示自動生成 → スタッフが仕入完了マーク → 発送指示自動生成 → 追跡番号入力 → eBay 反映
- Ship by Date は Fulfillment API から正確に取得（祝日テーブル不要）
- キャンセル・返金は自動検知して発送停止

詳細は `05-procurement/cases/shiire-tool.md`（社長運用中の仕入管理表）参照。

---

## メンバーへの含意

- **発送業務単独でAI化するより、仕入管理と一体化したほうが効率的**
- **eBay公式 API 活用が鍵**（Fulfillment API・Sell API）
- **倉庫スタッフのフィードバックが最も価値ある改善源**（運用してみて初めて分かる）
