# 09. 分析・KPI — 数字を見る・レポート作る業務のAI化

> このフォルダは「売上・KPI分析・週次／月次レポート作成」業務のAI化事例を集める棚です。

---

## この業務段階で何ができるか（メンバー向け）

### 課題（メンバーが抱えがちな悩み）
- eBayレポートのダウンロード → Excel 集計 → 分析 が毎週の手作業
- 数字を見ているのに「で、何をすべきか」が分からない
- 売れ筋・死蔵品の判断が感覚頼り
- 前週比・前月比の比較がしんどい
- レポートの書式が毎回バラバラ

### Claude Code でできること
- **eBay週次/月次レポート完全自動化**（API → 分析 → Excel/Sheets → Chatwork通知）
- **CVR・CTR・Impressions の自動集計**（業界平均比較込み）
- **売れ筋・死蔵品の自動分類**（Impression ゼロ・CVR低い等の閾値判定）
- **前週比・前月比の自動算出**
- **「で、何をすべきか」のアクション提案**（数字 → 施策の橋渡し）

## メンバーが最初に試すべきこと

### ステップ1：現状のレポート作業を棚卸し
- 週次/月次でどのレポートを作っている？
- 何を見て判断している？（CVR / 売上 / 利益率 / 在庫回転）
- 1回のレポート作成にどれくらい時間取られている？

### ステップ2：1つを完全自動化
- 一番時間のかかるレポートを選ぶ
- データ取得 → 集計 → 出力 → 通知 の流れを Claude に作ってもらう

### ステップ3：判断ロジックを言語化
- 「CVR < 0.5% は死蔵品候補」「前週比 -20% 以上は要確認」等
- これを自動判定に組み込む

## 社長の体験：eBay週次レポートv3（要点）

### 何をやったか
- eBay の3つのレポート（Traffic / Advertising Sales / Transaction）を統合
- API完全自動化（CSVダウンロード不要）
- Excel + Google Sheets に同時書込
- 毎週月曜10:00 に Chatwork へ自動配信
- 12シート構成（概要・売上ゼロ・売れ筋・広告・育成候補 等）

### 取り組みのプロセス
- v1: CSV手動ダウンロード → Python で集計
- v2: GetOrders API実装で CSV不要化
- v3: GSheets書込対応・前週比レポート完成
- 履歴データ重複バグの根本修正（HISTORY_DATE_KEY設計）

### つまづいたこと
- ハードコードヘッダーが多発・前週カテゴリ全部同じバグ
- merge制約エラー（freeze境界問題）
- COUNTIF の配列結合罠

### 学んだこと
- **3レポート統合で初めて「実態」が見える**（バラバラに見ると判断を誤る）
- **Transaction Report は USD のみ使う**（ebaymag は混ざると狂う）
- **PLP/Offsite は別途取得が必要**（Transaction には含まれない）

### 数値で見えた事実（社長運用例）
- 全出品 2,856件のうち**売上ゼロが2,647件（92.7%）**
- 全体CVR：0.221%（業界平均2〜3%の約1/10）
- 完全死蔵（Impressionゼロ・Views0・Sales0）：14件

→ こういう「衝撃の数字」が見えて初めて打ち手が打てる

### 数値 Before/After
- 手動運用：3〜4時間/週
- 自動化：完全自動・配信まで0秒

## メンバーへの含意

- **数字を見る習慣がない人ほど効果大**（眠っていた問題が見える）
- **業界平均との比較が判断材料**（自分の数字だけ見ても良し悪し分からない）
- **Chatwork等への自動配信で見る習慣化**（メールやファイルだと見ない）
- **「で、何をすべきか」まで踏み込む分析が価値**
