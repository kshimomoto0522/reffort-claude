## 何をやったか（目的・背景）

eBay の3つの主要レポートを統合し、週次で「で、何をすべきか」まで踏み込んだレポートを自動配信する仕組み。

社長は3,000件規模の出品を運用しているため、人間がExcelで集計するのは現実的でない。完全自動化により**毎週月曜の朝には新鮮な数字とアクション提案がChatworkに届く状態**を実現。

## つまづいたこと（失敗・遠回り）

### 失敗1：3レポートを別々に見て判断ミス
- Traffic Report のみで「インプ多いから優良商品」と誤判断
- Transaction Report で売上ゼロが発覚
- 教訓：**3レポート統合で初めて実態が見える**

### 失敗2：Transaction Report の通貨混在
- `Transaction currency = USD` と `ebaymag`（円建て）が混在
- ドル建て分析が狂った
- 教訓：**通貨フィルタは必須**

### 失敗3：PLP/Offsite を見落とし
- Transaction Report に含まれていない
- 別途 Seller Hub の広告管理画面で確認が必要
- 「利益が多く見える」状態が長期間続いていた
- 教訓：**広告は別系統で集計が必要**

### 失敗4：ハードコードヘッダーが多発
- v3 開発時にコード内に「列A=注文ID、列B=日付」等を直書き
- シート構造変更のたびに全箇所修正
- 教訓：**ヘッダーは定数管理**

### 失敗5：merge制約エラー
- Google Sheets API で freeze と merge を併用するとエラー
- freeze境界部分は merge 不可
- 教訓：**freeze 境界部分の制約を理解**

## 学んだこと（教訓）

- **3レポート統合で初めて「実態」が見える**（バラバラに見ると判断を誤る）
- **Transaction Report は USD のみ使う**（ebaymag は混ざると狂う）
- **PLP/Offsite は別途取得が必要**（Transaction には含まれない）
- **「で、何をすべきか」まで踏み込む**（数字を出すだけでは意味がない）
- **業界平均と比較**（自分の数字だけ見ても良し悪し分からない）
- **Chatwork等への自動配信で見る習慣化**（メールやファイルだと見ない）

## メンバーへの含意（応用ポイント） ★

### あなたの規模でも自動化は可能
- 月100件以上の運用なら効果大
- 月10〜50件規模でも、可視化することで「眠っている問題」が見える

### 必要な前提
- Python or Node.js で API を叩ける
- Google Sheets / Chatwork API の利用
- .env で認証情報を管理できる

### 最初の1個を作る手順
1. 自分の最重要KPI（売上 / CVR / 広告費率 等）を3つ決める
2. それを得るために必要なeBay API を特定（GetOrders / Marketing API 等）
3. Claude Code に「これらのデータを毎週月曜に集計して Chatwork配信して」と依頼
4. まず Excel/CSV出力から始める（最終目標はSheets書込）
5. 1ヶ月運用してアクション提案を肉付け

### やってはいけないこと
- ❌ 数字を出すだけで終わる（「で、何をすべきか」がない）
- ❌ 業界平均を知らずに自分の数字だけ見る
- ❌ 通貨フィルタなしで集計（ebaymag混入で狂う）
- ❌ レポートを誰も見ない場所に置く（メール添付・ローカルExcel）

## 衝撃の数字（社長運用例）

社長が初めて自動レポートを見て判明した事実：
- 全出品 2,856件のうち**売上ゼロが2,647件（92.7%）**
- 全体CVR：**0.221%**（業界平均2〜3%の **約1/10**）
- 完全死蔵（Impressionゼロ・Views0・Sales0）：14件
- 総収入 $64,600 に対し広告費合計 $6,336（広告費率約9.8%）

→ こういう「眠っていた問題」が見えて初めて打ち手が打てる。
→ 多くのセラーは見ていないだけで同じ状態にある可能性が高い。
