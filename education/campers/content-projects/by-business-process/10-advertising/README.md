# 10. 広告最適化 — eBay広告（PLG/PLP/Offsite）のAI化

> このフォルダは「eBay広告の運用最適化」業務のAI化事例を集める棚です。

---

## この業務段階で何ができるか（メンバー向け）

### 課題（メンバーが抱えがちな悩み）
- PLG（成果報酬型）に一律何%か設定したまま放置
- PLP（クリック課金）・Offsite の効果が分からない
- 広告費がいくら掛かっているのか実態を把握していない
- 売れ筋商品にも非売れ筋にも同じ広告料率を払っている
- 広告費率が利益を圧迫しているのに気付けない

### Claude Code でできること
- **PLG/PLP/Offsite の完全可視化**（広告費 vs 売上を商品単位で）
- **帰属売上の正確な分析**（重複計上・エラー検知）
- **階層型広告料率の自動最適化**（売れ筋2%・育成5%・死蔵停止）
- **Marketing API による完全自動管理**（手動調整不要）
- **「広告税」状態の発見と対処**（一律設定の弊害を数値化）

## メンバーが最初に試すべきこと

### ステップ1：現状の広告設定を確認
- PLG は一律何%？（多くの人が5%固定）
- PLP・Offsite はやっている？
- 過去3ヶ月の広告費合計を把握している？

### ステップ2：実態の可視化
- 広告費 vs 売上を商品単位で集計
- 「広告費が売上の○%を占めている」を数値化
- 売れ筋と死蔵品で広告料率が同じになっていないか

### ステップ3：階層型へ移行
- 売れ筋（過去30日売上あり）：2%（最低料率）
- 育成商品（インプ高だが売上低）：5〜8%
- 死蔵品（インプ低・売上ゼロ）：0% または取り下げ

---

## 社長の体験：eBay広告最適化プロジェクト（要点）

### 何をやったか（2026-03-26〜継続中）
- PLG/PLP/Offsite の3種類を完全可視化
- 帰属ルール変更（2026年1月）の見落とし発見
- Marketing API による自動化検討中

### 衝撃の発見
- **PLG一律5%が実質「広告税」状態**（売れ筋にも死蔵品にも同じ料率）
- **帰属売上 $68,963 / 総売上 $66,724 = 帰属率103%**（明らかに重複計上）
- **Offsite広告 2,878商品中、売却58商品のみ**（成果率2%）

### つまづいたこと
- 2026年1月の帰属ルール変更を約3ヶ月見落としていた
- 「広告費は仕方ないコスト」と思考停止していた

### 学んだこと
- **PLG階層型（売れ筋2%・育成5%）が推奨設定**
- **API による完全自動管理が可能**（Marketing API）
- **帰属率103%は誰にでも起きる**（重複計上の検知が必須）
- **「広告税」発想からの脱却**が利益率回復の鍵

### 数値で見えた事実
- 総収入 $64,600 に対し広告費合計 $6,336（**広告費率約9.8%**）
- これを階層型に移行できれば数千ドル/月の利益改善見込み

---

## メンバーへの含意

- **「広告費5%だから」と思考停止していないか**：階層型への移行で利益率改善余地大
- **eBayは数値が全て見える**：見ていないだけで実態は把握可能
- **帰属ルール変更は要注意**：年1〜2回の eBay 仕様変更を見逃すと損する
- **Marketing API で自動化可能**：手動調整は時代遅れ
