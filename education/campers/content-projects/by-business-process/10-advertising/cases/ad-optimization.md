# eBay広告最適化（PLG/PLP/Offsite・帰属103%発見・「広告税」エピソード）

> 業務段階: 10-広告最適化
> 状況: 社長運用継続中（2026-03-26 開始・最適化進行中）
> 元素材: journey-log.md L633-700

---

## 何をやったか（目的・背景）

eBayの3種類の広告（PLG/PLP/Offsite）の実態を完全可視化し、「一律設定」から「階層型最適化」へ移行するプロジェクト。

社長は数年間「PLG一律5%」のまま放置していた。ある日、改めて広告費の総額と帰属売上を可視化したところ、**衝撃の事実が次々と発覚**：

- PLG一律5%が実質「広告税」状態（売れ筋にも死蔵品にも同じ料率）
- 帰属売上 $68,963 / 総売上 $66,724 = **帰属率103%**（明らかに重複計上）
- Offsite広告 2,878商品中、**売却58商品のみ**（成果率2%）

このプロジェクトは社長の「数年来の思考停止」を打ち破った象徴的な事例。

---

## 取り組みのプロセス（時系列要点）

### 可視化フェーズ（2026-03-26）
- PLG / PLP / Offsite を別々に集計
- 商品単位の広告費 vs 売上をマッピング
- 「広告税」状態（一律料率の弊害）を発見
- 帰属率103%の異常を発見

### 分析フェーズ
- 2026年1月の帰属ルール変更を見落としていたことが発覚
- 重複計上が3ヶ月続いていた
- Offsite広告の効果が極端に低いことを定量化

### 最適化フェーズ（進行中）
- 階層型 PLG 設計：
  - 売れ筋（過去30日売上あり）：2%（最低料率）
  - 育成商品（インプ高だが売上低）：5〜8%
  - 死蔵品（インプ低・売上ゼロ）：0% または取り下げ
- Marketing API での自動管理を検討中

---

## つまづいたこと（失敗・遠回り）

### 失敗1：「広告費は仕方ないコスト」と思考停止
- 数年間 PLG 5% を疑わず放置
- 売れ筋にも死蔵品にも同じ料率
- 教訓：**「常識」を疑う・定期的に実態を可視化**

### 失敗2：帰属ルール変更を3ヶ月見落とし
- 2026年1月のeBay仕様変更通知を見ていなかった
- 帰属売上が103%（重複計上）の状態を3ヶ月放置
- 教訓：**eBay公式アナウンスは月次でチェック**

### 失敗3：Transaction Report だけで判断
- PLP/Offsite は別途取得が必要
- Transaction Report のみだと「利益が多く見える」状態
- 教訓：**広告は3系統別々に集計**

### 失敗4：Offsite広告を盲信
- 2,878商品が Offsite対象になっていた
- 実は売却したのは58商品のみ（成果率2%）
- 教訓：**「広告掛けっぱなし」は危険・定期見直し必須**

---

## 学んだこと（教訓）

- **PLG階層型（売れ筋2%・育成5%）が推奨設定**
- **API による完全自動管理が可能**（Marketing API）
- **帰属率103%は誰にでも起きる**：重複計上の検知が必須
- **「広告税」発想からの脱却**が利益率回復の鍵
- **Offsite は厳選**：闇雲に対象を広げない

## メンバーへの含意（応用ポイント） ★

### あなたも同じ状態かもしれない
- PLG一律5%のまま放置していませんか？
- Offsite広告を「とりあえず全商品オン」にしていませんか？
- 帰属率を一度でも計算したことがありますか？

### 必要な前提
- eBay Seller Hub にアクセス可能
- 過去3ヶ月の Transaction Report と Advertising Sales Report を持っている
- Excel / Sheets で集計できる（Claude Code が手伝う）

### 最初の1個を作る手順
1. Transaction Report と Advertising Sales Report をダウンロード
2. Claude に「商品単位で広告費と売上を集計して、広告費率の高い商品TOP20を出して」と依頼
3. 「広告費率 > 10% の商品」「Impressionゼロの商品」を可視化
4. 階層型料率の設計（売れ筋・育成・死蔵）
5. Seller Hub 上で手動変更（最初は10〜20商品で試す）
6. 1ヶ月後に効果検証
7. 効果が確認できたら Marketing API での自動化検討

### やってはいけないこと
- ❌ PLG一律5%のまま放置
- ❌ Offsite を「全オン」のまま
- ❌ 帰属ルール変更を見逃す（年1〜2回ある）
- ❌ Transaction Report だけで広告効果を判断
- ❌ いきなり全商品の料率変更（最初は10〜20商品で）
