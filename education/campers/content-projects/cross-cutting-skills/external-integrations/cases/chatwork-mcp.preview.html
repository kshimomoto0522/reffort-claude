Chatwork MCP 連携（自動配信・通知）

業務段階横断 状況: 社長運用継続中（複数タスクで毎日配信）

何をやったか（目的・背景）

Claude Code から Chatwork に自動でメッセージ送信・取得する仕組み。

社長は社内連絡を Chatwork に集約しているため、Claude Code が実行した結果を Chatwork に自動配信できれば情報が一元化される。

できること

  • 週次レポートの自動配信（毎週月曜10:00）
  • X情報ダイジェストの自動配信（毎日9:40）
  • 隔週メンテ結果の通知（第1・第3月曜10:00）
  • Campersメンバー削除完了通知
  • スケジュールタスクの実行結果通知
  • 社長個人DM・社内グループ・Campers部屋への投稿
  • DM内容の自動取得・要約

取り組みのプロセス

  1. Chatwork API トークン取得
  2. .env に保存（CHATWORK_API_TOKEN=xxx）
  3. MCP サーバー設定（mcp__chatwork__* ツール群）
  4. 投稿先 room_id を確認
  5. メッセージ整形ルール確立（[info][title]形式・箱型カード）

つまづいたこと

失敗1：誤ルームへの誤送信

  • room_id をハードコードしていて切り替え時にミス
  • 対策：room_id 一覧を memory で管理

失敗2：「To」漏れで通知されない

  • グループルームでメンションなしだと相手に通知が飛ばない
  • 対策：重要メッセージは必ず [To:user_id] を付ける

失敗3：箱型カードフォーマット未統一

  • 個人DM 426170119（社長個人）には [info][title] 形式の箱型カード必須
  • 対策：memory feedback_chatwork.md でルール固定

メンバーへの含意（応用ポイント） ★

あなたが使っているチャットツールでも同じことができる

  • Chatwork（社長の社内）
  • Slack（社長の Cowatech連絡用）
  • LINE Notify（簡易通知に便利）
  • Discord（コミュニティ運営なら）

必要な前提

  • 該当チャットツールの API トークン取得
  • .env での認証情報管理
  • MCP サーバーの設定

最初の1個を作る手順

  1. Chatwork API トークン発行（Chatwork 設定 > API）
  2. .env に保存
  3. Claude Code の MCP 設定に Chatwork を追加
  4. テスト：「Chatworkにテストメッセージ送って」と依頼
  5. 自動配信したい業務（週次レポート等）に組み込み

やってはいけないこと

  • ❌ APIトークンをコード直書き
  • ❌ 確認なしで重要メッセージ送信（送信前にGO確認）
  • ❌ 全ルームへの誤送信（room_id を必ず確認）
