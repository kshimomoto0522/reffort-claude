5/31 Campersウェビナー骨子（v0.1 枠組み）

日時: 2026年5月31日（日） 対象: Campersメンバー（約40名・eBayセラー・非エンジニア） テーマ: eBay × AI運営 — あなたの業務もあなた自身でAI化できる 話者: 下元 敬介（株式会社リフォート 代表取締役）

⚠️ 本ファイルは枠組みドラフト。4/26版（education/consulting/webinar-draft-20260426.md）をベースに、コンテンツ蓄積基盤整備（2026-04-29）の方針を反映した v0.1。実際のスクリプト肉付け・スライド作成は社長との対話で進める。

核心メッセージ（再掲）

「あなたもプログラミング未経験でも、Claude Code で自分の業務をAI化できる。私が体験した変化を、あなたの商材・あなたの業務で実現するロードマップを示す」

社長の事例（在庫管理ツール／週次レポート／広告最適化／仕入管理 等）は「テンプレ」ではなく「方法論を例示する素材」として配置する。

全体構成（約90分想定）

パート 内容 時間
1 共感パート — メンバーのリアルを言葉にする 5分
2 変化のストーリー — 私（社長）が体験した3ヶ月 15分
3 方法論 — eBay運営11業務段階の AI化マトリクス 25分
4 第一歩 — あなたの最初の1個を選ぶフレームワーク 15分
5 「Claude Code はeBay以外でも」— 自社サービスの参考例 10分
6 AIコース月額の案内 — 伴走の提案 10分
7 Q&A 自由

パート1：共感パート（5分）

狙い

メンバーの今のリアルを言葉にして、共感のフックを作る。「自分のことを言われている」と感じてもらう。

トピック

  • トランプ関税の影響でeBayが厳しい
  • Campers も70名→40名に減少している事実
  • 売上ダメージ・新しい打ち手が見つからない・時間が足りない
  • 「同じやり方を続けているだけでは生き残れない」

使う素材

  • 4/26版 webinar-draft-20260426.md パート1 を流用・更新

注意

  • 「皆さんも同じ状況ですよね？」と巻き込む
  • 自分の数字（売上 $60,000 等）公開可否は社長判断

パート2：変化のストーリー（15分）

狙い

社長自身が「プログラミング未経験」から「Claude Code で人生変わった」までの数字＋エピソードで、変化の現実性を示す。

トピック

  • 2026年3月11日にClaude Code導入
  • 1ヶ月半でeBay運営のやり方が根本から変わった
  • 業務効率化を超えた変化（経営パートナーAIを持つ感覚）
  • 数字での裏付け：時間短縮・コスト削減・利益率改善

使う素材（業務縦軸から抽出）

  • 00-claude-code-foundation/：Claude Code を入れて何が変わったか
  • 09-analytics/cases/weekly-report.md：週次レポート手動3〜4時間 → 完全自動の Before/After
  • 05-procurement/cases/shiire-tool.md：仕入忘れ月数件 → ゼロ
  • 10-advertising/cases/ad-optimization.md：「広告税」発見・帰属率103%

注意

  • 数字公開範囲は社長判断
  • 「私だからできた」ではなく「方法論があれば誰でもできる」と伝える

パート3：方法論 — eBay運営11業務段階の AI化マトリクス（25分・最重要）

狙い

「あなたの業務もAI化できる」を体系的に示す。商材バラバラのメンバー全員が「自分の業務に当てはめられる」状態を作る。

構成

  1. eBay運営の業務プロセス全体図を提示（10業務段階＋Claude Code基盤）
  2. 各業務段階で「AIで何ができるか」を概要紹介
  3. 自分（社長）の事例は「業務プロセスのこの段階で、私はこうした」と位置づけて紹介

11業務段階の紹介順（推奨）

  1. 00-claude-code-foundation（土台）→ メンバー向け導入の入口
  2. 09-analytics（数字を見るインパクト・92.7%売上ゼロの衝撃）
  3. 10-advertising（広告税エピソード）
  4. 05-procurement（仕入管理AI化の代表例）
  5. 03-mukozai-stock-management（リサーチ・スクレイピング系）
  6. 04-uzaiko-stock-management（有在庫の人向け）
  7. 08-accounting（スタッフがいる人向け）
  8. 01-research / 02-listing（5/31までに事例作成・予告）
  9. 07-shipping / 11-direct-sales（簡潔に）
  10. 06-trouble-handling（参考までに）

使う素材

  • 各業務段階の README.md
  • 各業務段階の cases/*.md
  • INDEX.md の業務縦軸ビュー

注意

  • 1業務段階あたり2〜3分で簡潔に
  • 詳細は AIコースで深掘りすると伝える

パート4：第一歩 — あなたの最初の1個を選ぶフレームワーク（15分）

狙い

メンバーが自分の業務に重ねて「最初に何をAI化するか」を決められる状態にする。

フレームワーク

  1. 業務棚卸し：自分の1日／1週間の業務を11業務段階に当てはめる
  2. ボトルネック特定：どこが最も時間取られているか
  3. AI化優先度を1〜5で記入：CLAUDE.mdテンプレ（03_claude-md-template.md）参照
  4. 最初の1個を選ぶ基準：
  5. 毎日／毎週やっている定型作業
  6. 数値で効果が測れるもの
  7. 失敗してもダメージが小さいもの

使う素材

  • 00-claude-code-foundation/cases/03_claude-md-template.md の業務プロセス優先度表
  • 00-claude-code-foundation/cases/07_complete-setup-checklist.md

配布素材（候補）

  • 業務棚卸しワークシート（PDF）
  • 11業務段階の早見表

パート5：Claude Code はeBay以外でも（10分）

狙い

「Claude Code を使いこなせばeBayだけでなく、他事業にも・新規事業にも応用できる」を示す。

トピック

  • 自社サービス（BayChat）でも Claude Code を活用していること
  • AI Reply のプロンプト改善サイクル
  • 外注先（Cowatech）への発注書も Claude が支援
  • ダイレクト販売サイトの構築も可能

使う素材

  • applications-beyond-ebay/cases/baychat-ai-reply.md（参考例として）
  • 11-direct-sales/cases/direct-sales.md（直取応用）

重要な注意

  • BayChat の話は Campers向け参考例のみ（匿名X／Note には出さない）
  • 「皆さんが新規事業を考えるときも応用できる」というメッセージ

パート6：AIコース月額の案内（10分）

狙い

単発講座で終わらせず、メンバーが自分の業務をAI化するまで伴走する月額コースを案内する。

提案するコース内容（社長確認待ち）

  • 月額追加（Campers月会費に上乗せ・金額は社長判断）
  • 月次の課題・Q&A・個別相談・ツール開発サポート
  • ロードマップ：業務棚卸し → ボトルネック特定 → 最初の1個 → スプレッドシート活用 → API連携 → 自動化 → 自分専用ツール
  • 詳細は ai-course-curriculum/curriculum-draft.md 参照

注意

  • 「金を巻き取る」感を出さない
  • 「メンバーに新たな収入源を作る・スキルアップ・脱退抑止」が動機
  • 「金を払ってでも学びたい」と感じてもらえる中身が前提

パート7：Q&A

想定される質問

  • 「私の取扱商材でも本当にできるのか？」→ 商材問わず業務プロセスは共通
  • 「プログラミング未経験で大丈夫か？」→ 社長自身が未経験から始めた
  • 「コストはどれくらいかかるのか？」→ Claude Maxプラン＋APIで月数万円程度
  • 「最初の1個に何を選ぶべきか？」→ 業務棚卸し → ボトルネック特定で決まる
  • 「失敗が怖い」→ 小さく始める・失敗込みで学ぶ
