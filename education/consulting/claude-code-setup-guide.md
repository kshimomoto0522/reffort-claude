## このガイドで学べること

- Claude Codeをゼロからインストールして使えるようにする
- 自分のビジネス専用の「記憶」を持つAIアシスタントを構築する
- MCPで外部ツール（Chatwork・Slack等）と連携する
- 定期実行タスクを自動化する
- 複数PC・スタッフで使う環境を整える

## Unit 1：インストールと初回起動

### 1-1. アカウント作成
1. `claude.ai` にアクセスしてAnthropicアカウントを作成
2. プランを選択
   - **Proプラン（$20/月）**：個人利用・1セッションで十分な場合
   - **Maxプラン（$100/月〜）**：複数エージェント並行実行・大量タスクに向く
   - 💡 **下元の選択**：最初はProで開始し、後にMaxに変更。Maxにしたら「2つのエージェントを並行で走らせます」と表示されるようになった

### 1-2. Claude Codeのインストール（Windows）
1. `claude.ai/download` にアクセス
2. Windows版インストーラーをダウンロード
3. インストール後、同じAnthropicアカウントでログイン

### 1-3. 初回起動時の確認
- 上部に「チャット」「Cowork」「コード」タブがある
  - **チャット** = 会話画面（メインで使う）
  - **コード** = ファイル編集画面
  - **Cowork** = 他のユーザーをリアルタイムで招待して共同作業できる
- 左サイドバー = 過去のセッション一覧（ブラウザの履歴のようなもの）
- 左下 = 現在開いているフォルダのパスが表示される

### 1-4. フォルダを開く
Claude Codeは「どのフォルダで起動するか」が重要。フォルダごとに別の「指示書（CLAUDE.md）」を読み込む。

## Unit 2：CLAUDE.mdの作り方

### 2-1. CLAUDE.mdとは
- Claude Codeが起動するたびに自動で読み込む「指示書」
- ここに会社・事業・ルール・スタッフ情報を書いておくと、毎回説明しなくてよくなる
- フォルダごとに置くことで「部門専用Claude」として動かせる

### 2-2. フォルダ構成の設計（下元の実例）
```
/reffort/                    ← 本社フォルダ（全社共通の指示書）
  CLAUDE.md                  ← 会社全体の記憶
  /commerce/ebay/analytics/           ← eBay分析部門
    CLAUDE.md                ← eBay専用の指示書
  /services/baychat/ai/               ← BayChat AI部門
    CLAUDE.md
  /services/baychat/marketing/                ← マーケ部門
    CLAUDE.md
  /commerce/ebay/staff-ops/                ← スタッフ管理部門
    CLAUDE.md
  /management/               ← 経営管理部門
    CLAUDE.md
  /education/campers/                  ← スクール部門
    CLAUDE.md
  /education/consulting/               ← コンサル事業準備部門
    CLAUDE.md
    journey-log.md           ← 実践記録（このファイル群）
```

### 2-3. CLAUDE.mdに書くべき内容
```markdown
# 会社名 - 部門名

## 事業概要
（会社・事業の基本情報）

## スタッフ体制
（誰が何をやっているか）

## 使用ツール
（スプレッドシート・外部サービス等）

## 現在の課題
（今一番困っていること）

## Claudeへの重要ルール
（この部門でClaudeに必ず守らせること）
```

### 2-4. ⚠️ 失敗談：仮情報でCLAUDE.mdを作ってはいけない
最初、Claudeがマスタードキュメントの情報だけで各部門のCLAUDE.mdを自動作成した。しかし内容が「仮情報」だったため、後で全部作り直すことになった。

**教訓：CLAUDE.mdはQ&A形式で正確な情報を集めてから作る。急がば回れ。**

### 2-5. 実際の作り方手順
1. `reffort`フォルダに`CLAUDE.md`を新規作成
2. Claudeに「私の会社について質問して、CLAUDE.mdを作って」と依頼
3. Claudeが1部門ずつ質問してくる → 答える → 作成される
4. 作成後に読んで「この情報が違う」を修正する

## Unit 3：メモリ（長期記憶）の仕組み

### 3-1. メモリとは
- セッションをまたいで記憶を保持するファイル群
- 場所：`C:\Users\[ユーザー名]\.claude\projects\...\memory\`
- CLAUDE.mdと違い、「会話から自動的に学んだこと」を保存する

### 3-2. メモリの種類
| 種類 | 内容 |
|------|------|
| user | ユーザーのスキル・目標・好み |
| feedback | 「こうしてほしい」「これはダメ」という指示 |
| project | 進行中のプロジェクト状況 |
| reference | 外部システムの参照先 |

### 3-3. 保存タイミング
- 「メモっておいて」と言えば即保存
- Claudeが「これは覚えておくべき」と判断したら自動保存
- セッション終了時（「今日はここまで」等）に自動チェック

## Unit 4：セッション管理

### 4-1. セッションとは
- 1つの会話のこと
- 左サイドバーに過去のセッションが一覧表示される

### 4-2. 新しいセッションの開始
- 左上「+ 新規セッション」をクリック
- 同じフォルダで開始 → 同じCLAUDE.mdを読む → すぐに文脈が戻る

### 4-3. セッションが重くなったら
- 会話が長くなるとレスポンスが遅くなる・精度が落ちる
- **対処法：新しいセッションを開始するだけ**でOK。CLAUDE.md+メモリが引き継がれる
- Claudeから「そろそろ新しいセッションに引き継ぎますか？」と声をかけるルールを設定済み

### 4-4. 別部門のセッションを開く（マルチセッション）
- Claude Codeアプリをもう1つ起動する
- 別のフォルダ（例：`services/baychat/ai`）を開く
- 2つのセッションは完全に独立して動く

### 4-5. セッション名の変更
- セッション名を変えても内容・CLAUDE.md・メモリに影響なし
- 自由に変えてよい

## Unit 5：MCP（外部ツール連携）の設定

### 5-1. MCPとは
Model Context Protocolの略。Claude Codeと外部ツールを繋ぐ仕組み。

**MCPを設定すると何ができるか：**
- Chatworkにメッセージを送る
- Slackを読み書きする
- Googleスプレッドシートを操作する
- ブラウザを操作する

### 5-2. Chatwork MCPの設定（下元の実例）
1. Chatworkの設定画面からAPIトークンを取得
2. Claude CodeのMCP設定にChatwork MCPを追加
3. 送信テストを実施

💡 **ポイント：APIトークンは絶対にコードに直書きしない**
→ `.env`ファイルまたは設定ファイルで管理する（セキュリティルール）

### 5-3. MCPで実現した自動化（下元の実例）
- 毎週月曜10:00にeBay週次レポートExcelをChatworkグループに自動配信
- 前週比コメント付き（受注数・売上・CVR）

## Unit 6：スケジュールタスクの設定

### 6-1. スケジュールタスクとは
決まった時間に自動でClaudeに指示を実行させる仕組み。

### 6-2. 設定方法
Claude Codeに「毎週月曜10時に〇〇を実行して」と依頼するだけ。
内部的にはタスクファイルが生成される。

### 6-3. 下元が設定したタスク
| タスクID | 内容 | スケジュール |
|----------|------|------------|
| `daily-github-backup` | CLAUDE.mdファイルをGitHubに自動バックアップ | 毎日深夜0時 |
| `monday-ebay-report-delivery` | eBay週次レポート生成→Chatwork自動配信 | 毎週月曜10:00 |

### 6-4. ⚠️ 注意点
- タスクはPC起動中のみ実行される
- 実行時間はJST（日本時間）で設定すること

## Unit 7：複数PC・スタッフへの展開

### 7-1. 別のPCでClaude Codeを使う手順
1. 新しいPCにClaude Codeをインストール（`claude.ai/download`）
2. 同じAnthropicアカウントでログイン
3. `reffort`フォルダを移動する（方法は以下）

**フォルダの移動方法：**

| 方法 | 特徴 |
|------|------|
| USBメモリ | 今すぐできる。ただし都度同期が必要 |
| OneDrive/Googleドライブ | クラウド経由で自動同期 |
| **GitHub（推奨）** | 完全自動同期・履歴管理・複数人で使える |

⚠️ **メモリファイルは別PCに引き継がれない。** CLAUDE.mdは引き継がれるので基本的な文脈は戻るが、細かい記憶は薄くなる。

### 7-2. GitHubでの同期（最終推奨）
- GitHubにリポジトリを作成
- `reffort`フォルダを紐付け
- 変更するたびに自動でバックアップ・同期される
- 複数PCから同じ状態でアクセス可能
- ※設定は約10分。プログラミング不要。

### 7-3. スタッフにClaude Codeを使わせる場合の選択肢

| アプローチ | 向いているケース | コスト |
|-----------|----------------|--------|
| 各自がアカウントを持つ | 日常的に使うスタッフ（マーケ担当等） | 月$20/人 |
| 社長PCで作業させる | たまにしか使わない・スポット作業 | 0円 |
| Coworkで招待 | リアルタイム共同作業 | 要アカウント |
| Claudeの出力をChatworkで共有 | 現場スタッフ（eBay運営等） | 0円 |

### 7-4. 承認フローの考え方
- スタッフが作業 → GitHubで「プルリクエスト（変更申請）」を出す
- 社長が内容を確認してOKなら取り込む
- GitHubバックアップ設定時に一緒に組める

## Unit 8：よくある質問・詰まりポイント

**Q: セッションをまたいだら前の話を覚えていない**
→ CLAUDE.mdとメモリが読み込まれれば基本的な文脈は戻る。重要な決定事項はメモリに保存する習慣をつける。

**Q: レスポンスが遅くなった**
→ セッションが重くなっている。新しいセッションを開始するだけでOK。

**Q: ファイルを作成しようとしてエラーが出た**
→ 存在しないファイルにいきなり書き込もうとしている。先に空ファイルを作ってからWriteする順序が必要。

**Q: 上部の「チャット/Cowork/コード」タブは何？**
→ セッションの切り替えではなく、同じセッション内の表示モード切り替え。

**Q: セッション名を変えたら何か影響ある？**
→ ない。ラベルを変えるだけ。

**Q: アーカイブするとセッションが消える？**
→ 消えない。非表示になるだけ。戻せる。

**Q: ProプランとMaxプランの違いは？**
→ Maxにすると「複数エージェント並行実行」が使えるようになる。大量タスク・複雑な作業に向く。
