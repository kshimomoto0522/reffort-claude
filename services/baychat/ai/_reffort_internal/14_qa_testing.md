## 📐 UI仕様詳細（`UI.csv` 由来・ベトナム語原文要点）

### Admin画面の補足機能
セラーがAI Replyに入力できる補足情報（description）の入力フィールド。AIが受信する情報に加えて、バイヤーへの返信文を完成させる。

### Seller画面（AI返信画面）の構成

```
[AI返信サポート画面]
├─ Input 1: セラー補足（AI補完不可部分の手入力）
├─ Input 2: 日本語出力（jpnLanguage）
├─ Input 3: 選択言語出力（buyerLanguage）
├─ 言語選択: デフォルト=バイヤー言語 / オプション=英語
├─ Tone選択: [丁寧] / [フレンドリー] / [謝罪]
├─ 生成ボタン「返信を生成」: AIへのリクエスト
├─ Loading: BayChatロゴのアニメーション
└─ 送信ボタン「送信」: Input 3の内容を送信
```

### レスポンシブ動作
- 小画面：Input 1/2/3 が縦積みで縮小表示
- スマホ：tone icon は左右均等配置（gap固定不可）

## 🔄 新機能追加時のQAフロー

新機能（例：要約モード、新トーン追加）を実装する際に実施するチェック：

### フェーズ1：Cowatech実装前
- [ ] 設計図の該当ブロックカード更新
- [ ] 条件分岐表に新分岐を追記
- [ ] UI注入マトリクスに新UI項目を追記
- [ ] 変更履歴に変更理由と影響範囲を記録
- [ ] Cowatech仕様書（日本語）作成

### フェーズ2：Cowatech実装後
- [ ] ペイロード構造が設計図通りになっているか確認
- [ ] 動的プレースホルダ置換の動作確認
- [ ] 条件分岐の動作確認（OFF分岐テスト含む）
- [ ] UI表示の仕様整合性確認
- [ ] `testing/payload_builder.py` の更新

### フェーズ3：本番反映前
- [ ] 本番同等テスト実行（`batch_test.py --production-payload`）
- [ ] 品質スコアの劣化がないか確認
- [ ] コスト・レイテンシの変化が想定内か確認
- [ ] ロールバック手順の確認

## 🛡️ デグレ防止チェック（プロンプト変更時）

admin_prompt のバージョンアップ時に毎回実施：

1. **12ケース標準テスト**（`extracted_*.json` のベースラインセット）
2. **平均スコアが前版を下回らないこと**を確認
3. **カテゴリ別（cancel/return/claim/tracking/discount/general）のスコア分布**を確認
4. **本番モード（ON/OFF）両方でテスト**
5. **Excel結果とHTML比較レポートを社長レビュー**

## 🎨 UI設計原則（BayChat UI設計原則・memory由来）

### 配色
- **紫ベース**（BayChatブランドカラー）
- **信号機色（緑/赤/黄）禁止**
- 成功・警告・エラーは紫系の濃淡で表現

### 言語
- **英日併記必須**（UIコンポーネント）
- 例：「返信を生成」→「Generate Reply / 返信を生成」

### デザイン思想
- **シンプル・イズ・ザ・ベスト**
- 過剰な装飾・情報密度を避ける
