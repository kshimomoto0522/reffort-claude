05. 変更履歴

このファイルの役割：AI Replyに関わる全変更を時系列で記録する。
「いつ・誰が・なぜ・何を変えた・どこに影響した」の5項目を必ずセットで残す。
これによって、将来「なぜこの仕様なのか」を過去に遡って確認できる。

📝 記録フォーマット

変更を記録するときは、以下のテンプレートをコピーして先頭に追記する。新しい変更が上に来るようにする。

---

### [YYYY-MM-DD] 変更タイトル（短く）

| 項目 | 内容 |
|------|------|
| 変更対象 | （どのファイル/ブロック/パラメータ） |
| 変更者 | （社長 / Claude / Cowatech / 複数） |
| 変更理由 | （なぜ変えたか） |
| 変更内容 | （何を、どう変えたか） |
| 影響範囲 | （どこに波及するか） |
| テスト | （どう検証したか。未検証の場合はその旨） |
| 関連ドキュメント | （関連するブロックカード・handoff等のリンク） |
| ロールバック手順 | （問題があった時の戻し方） |

**本文（詳細・背景・議論のメモ）:**
詳細な説明や議論の背景、意思決定のプロセスをここに書く。後から読んでも判断理由が分かるように。
