15. 要約モード（Summary Mode）仕様

✅ v0.2 完成版（ドラフト段階）
このファイルの役割：将来機能「要約モード」の設計仕様をドキュメント化。現時点はドラフト（prompt_summary_v0.1.md）段階。

🎯 概要

要約モード は、AI Reply の次フェーズとして実装予定の新機能：
- 長文・複雑なバイヤーメッセージを 5要素構造で要約
- セラーが選択できる 対応パターン（3〜5個）を自動生成
- それぞれのパターンにメリット/デメリット・推奨度・理由を付与
- セラーは要約を見て方針を決め、その後 AI Reply で返信生成

🔀 Call-1 / Call-2 の構造

【Call-1：要約モード】← このファイルが定義
   入力：商品情報 + チャット履歴 + バイヤー最新メッセージ
   出力：
     ├─ summary（5要素で要約）
     ├─ complexityLevel（low/medium/high）
     ├─ patterns（3〜5個の対応案）
     └─ recommendedPattern（推奨ID）

       ↓ セラーがパターンを選択

【Call-2：返信生成（将来実装）】
   入力：選択されたパターン + その他情報
   出力：従来のAI Reply（jpnLanguage + buyerLanguage）

📐 要約モードのJSON出力スキーマ

{
  "summary": {
    "buyerEmotion": "感情ラベル+強度（1-5）",
    "factsStated": ["バイヤーが述べた客観事実の箇条書き"],
    "requestMade": "明示的な要望",
    "constraints": "制約条件（期限・金額・配送等）",
    "hiddenContext": "翻訳の不自然さ・暗黙の意図"
  },
  "complexityLevel": "low | medium | high",
  "patterns": [
    {
      "id": "A",
      "name": "対応パターン名",
      "description": "セラーが取る行動",
      "merits": ["利点1", "利点2"],
      "demerits": ["欠点1", "欠点2"],
      "recommendation": 1-5,
      "recommendationReason": "なぜその評価か"
    }
  ],
  "recommendedPattern": "パターンID",
  "sellerDecisionNeeded": "セラーが決めるべき核心（1行）"
}

🎚️ COMPLEXITY LEVEL の判定基準

Level 判定基準 AI Reply対応
low 単純な質問、1要件、感情穏やか 通常AI Reply で十分。要約モード不要
medium 2〜3要件混在、軽度な不満、期限あり 要約モード推奨
high 長文クレーム、複数要件混在、感情4以上、法的リスク、翻訳で意図不明瞭 要約モード必須

📋 PATTERNS生成ルール

  • 3〜5個を生成（少なすぎ=選択肢不足、多すぎ=判断疲労）
  • 各パターンは 方針が明確に異なる もの
  • ❌ 表現違いはNG（「返金」「全額返金」「完全返金」は同方針）
  • ✅ 方針違いはOK（「全額返金」「部分返金で継続」「代替品送付」）
  • merits / demerits：最大3個・体言止め・箇条書き
  • recommendation：1-5の整数（5が最推奨）
  • recommendationReason：1文

判断基準（AIに指示する評価軸）

  • eBayポリシー遵守
  • 評価（FB）リスク最小化
  • セラーの損失最小化

🚫 禁止事項

  • バイヤー向けの返信文を生成しない（Call-2の役割）
  • eBayポリシー違反の提案（Encourage off-platform等）
  • 推測と事実の混在（必ず「推測：」と明示）
  • 5要素いずれかを空文字で埋めない（該当なしなら「該当なし」と明記）
