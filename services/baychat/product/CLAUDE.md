## UI/UXデザインの進め方

### ワークフロー
```
① 社長がBayChatの画面スクショを共有
       ↓
② 修正・改善したい内容を伝える
       ↓
③ ClaudeがHTMLモックアップを作成 → プレビューで確認
       ↓
④ 社長が確認 → フィードバック → 修正を繰り返す
       ↓
⑤ 確定後、Cowatech向け仕様書 + モックアップを作成
       ↓
⑥ Slackでクエットさんに共有（送信ルールは /services/baychat/ai/CLAUDE.md に準拠）
```

### モックアップの形式
- **HTML/CSSプロトタイプ**: ブラウザでプレビュー可能、インタラクティブな動作確認もできる
- **Cowatech仕様書**: 日本語で「機能概要 / 変更点 / 画面イメージ / 注意事項」を記載

### モックアップの共有方法（徹底すること）
- **GitHub Pagesで公開し、URLをSlackに貼る**
- ベースURL: `https://kshimomoto0522.github.io/reffort-claude/`
- モックアップHTMLをGitHubにpush → 自動的にURLでアクセス可能になる
- クエットさんも社長もブラウザでそのまま確認できる（ダウンロード不要）
- GitHub Pagesは有効化済み（master / root）

## Claudeへの重要ルール（baychat-product部門専用）

- UIモックアップは必ずプレビューで社長に確認してもらってから仕様書化する
- 社長が「これでいい」と言うまで修正を繰り返す
- Cowatechへの仕様書は「この通りに実装してください」と言える精度で作る
- 機能の最終判断・優先順位は社長が決める
- 既存機能の動作確認にはスクショやテキストでの情報提供を社長に依頼する
- Slackでクエットさんに送信するときは必ずメンション（`<@U04HGPBABQU>`）をつける
- モックアップの共有はGitHub Pages経由のURLを徹底する（ファイル添付ではなくURL共有）
- 技術的な確認を社長に取るときは「なぜ確認が必要か」を簡単に説明した上で許可を求める（技術用語だけで聞かない）
